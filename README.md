# 高价值专利侵权检测

基于 Claude Code skill 的专利侵权排查工作区。给一个专利，自动产出一条"谁可能在用它"的分析报告链。

## 用法

在 Claude Code 中，直接给出专利号 / PDF 路径，或说一句触发语即可，例如：

- `检测 CN114531405B 有没有被其他公司使用`
- `排查这个专利的侵权情况`（附上 PDF）

skill（`patent-infringement-check`）会自动触发，按 7 步流水线把结果写进 `专利集/<专利号>/`。

## 产出（7 步）

1. `<专利号>.md` — 专利原文
2. `…"潜在应用场景及侵权特征".md` — 应用场景 + 侵权特征
3. `…"潜在使用组织".md` — 可能使用该技术的组织
4. `…"潜在侵权产品初选".md` — 各组织的具体产品
5. `…"潜在侵权产品-全".md` — 补充长尾后的完整候选列表
6. `候选/NN-<slug>/_verdict.md` — 每个候选的取证 + 合议结论
7. `…"违约列表".md` — 按档位排序的最终清单

## 证据复核

每个候选目录 `候选/NN-<slug>/` 是一条独立证据链，核查三点：

- **`_sources.md`** — 检索 query 留痕是否真实，来源 URL 的时间与类型（一手 / 二手）。
- **`_verdict.md`** F# 命中表 — verbatim 是否原文摘录、URL 是否可达、**反向证据 vs 限定作用域语**是否区分清楚。
- **最终档位** — 与 F# 分布是否自洽（**0 命中 ≠ 已排除**）。

目录内 PDF/HTML/源码为落盘原始材料，可直接比对原文。

## 注意

- 输出仅为线索与证据链，**不构成法律意见**，每份最终报告含免责声明。
- 只统计专利授权日之后公开的材料。
- skill 定义见 [.claude/skills/patent-infringement-check/SKILL.md](.claude/skills/patent-infringement-check/SKILL.md)，详细约定见 [CLAUDE.md](CLAUDE.md)。
