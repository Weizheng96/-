# CN104702691B 潜在应用场景及侵权特征

## 基本信息
- 专利号：CN104702691B
- 标题：分布式负载均衡方法和装置
- 当前权利人：华为技术有限公司 (Huawei Technologies Co Ltd)
- 申请日 / 公开（授权）日：2015-03-13 / 2017-12-01

## 一、潜在应用场景（严格来自背景技术 — 不外扩）

### 场景 1：分布式存储 / 缓存集群的自适应负载均衡
- **背景技术 verbatim 依据**：> "分布式系统中的数据可以分成多个数据分片，每个数据分片称为一个分区。分布式系统中的一个节点可以拥有一个或多个分区。……分布式系统的负载均衡问题就转化为所有主分区和备分区在集群节点上均匀分布的问题。"
- **场景说明**：分布式存储系统（有总控节点或无总控节点的 P2P 架构）的集群中，存储节点各自承载多个主分区（主副本）和备分区（备副本），需要动态调整分区分布以避免单点瓶颈。本专利正是解决主分区均匀分配 + 备分区均匀分配 + 备分区按负载迁移的整体方案。
- **使用本专利的典型形式**：集群管理模块 / 总控节点中的分区分配算法，或去中心化集群中 master 节点执行的负载均衡策略

### 场景 2：分布式系统的弹性扩缩容（scale-out / scale-in）
- **背景技术 verbatim 依据**：> "分布式系统除了提供更强的处理能力，还需要能够弹性扩展和具有高可靠性。弹性扩展的能力就是集群可以灵活的scale-out(扩容)和scale-in(缩容)。"
- **场景说明**：当分布式集群增加或移除节点时，需要重新均衡主 / 备分区分布，同时尽可能减少数据迁移量。
- **使用本专利的典型形式**：集群扩容 / 缩容后的自动负载重均衡模块

### 场景 3：分布式系统节点故障恢复与容错
- **背景技术 verbatim 依据**：> "现有方案单点故障后数据负载将被多个节点分担，但是比较难保证数据负载被其他节点完全均匀分担……当集群单点故障或缩容后，为保证数据均匀分布，需要重新调整主分区在各个节点上的分布情况，存在大量数据迁移操作。"
- **场景说明**：当集群中某节点发生故障后，该节点上的主分区对应备分区升级为主分区，然后需要将故障节点的副本重新均衡分配到剩余节点。
- **使用本专利的典型形式**：故障检测 + 备分区升级 + 分区重分配模块

## 二、独立权利要求 + 侵权特征 F1-Fk

### 权 1 verbatim
> 1.一种分布式负载均衡方法，应用于分布式系统，其特征在于，所述分布式系统的集群中包括多个节点和多个主分区，每个主分区对应于至少一个备分区，所述方法包括：
> 在所述多个节点上分配所述多个主分区，使得任意两个节点上的主分区个数差值小于第一预定阈值；
> 将所述多个主分区中的第一主分区对应的第一备分区分配到多个第二类节点中，使得所述第二类节点中任意两个节点上所述第一备分区个数之差都小于第二预定阈值，其中，所述第二类节点为所述第一主分区中第一类节点以外的节点，所述第一类节点为所述第一主分区所在的节点，或所述第一主分区所在的节点所属的分组的节点；
> 将所述多个第二类节点中的高负载节点的至少一个所述第一备分区移动到所述多个第二类节点中的低负载节点上，其中，所述高负载节点的分区个数大于第三预定阈值，所述低负载节点的分区个数小于第四预定阈值。

### F1-Fk 清单

- **F1（多节点多主分区集群）｜[AND]**：所述分布式系统的集群中包括多个节点和多个主分区，每个主分区对应于至少一个备分区
  - 可能描述方式：集群包含 N 个节点和 M 个主分片，每个主分片有 ≥1 个副本；分布式系统由多个服务器节点组成，数据划分为若干主分区且各有副本

- **F2（主分区均匀分配）｜[AND]**：在所述多个节点上分配所述多个主分区，使得任意两个节点上的主分区个数差值小于第一预定阈值
  - 可能描述方式：将主分片均匀分布到所有节点，使各节点主分片数之差 ≤1（K1=1）；round-robin 分配主分区后再做均衡调整；保证任意两节点主分区数之差不大于 K

- **F3（备分区分配到第二类节点 + 均匀约束）｜[AND]**：将所述多个主分区中的第一主分区对应的第一备分区分配到多个第二类节点中，使得所述第二类节点中任意两个节点上所述第一备分区个数之差都小于第二预定阈值，其中，所述第二类节点为所述第一主分区中第一类节点以外的节点，所述第一类节点为所述第一主分区所在的节点，或所述第一主分区所在的节点所属的分组的节点
  - 可能描述方式：将某主分区的副本分配到不包含该主分区所在节点的其他节点上，使这些节点上的该副本数 ≤1（K2=1）；备分区不能与对应主分区在同一节点 / 同一机架；将副本分配到不同的故障域；对于同一主分区的副本，在其非宿主节点间均匀分布

- **F4（备分区按负载移动）｜[AND]**：将所述多个第二类节点中的高负载节点的至少一个所述第一备分区移动到所述多个第二类节点中的低负载节点上，其中，所述高负载节点的分区个数大于第三预定阈值，所述低负载节点的分区个数小于第四预定阈值
  - 可能描述方式：将高负载节点上的副本迁移到低负载节点；节点总分区数超过上限时移出副本到分区数低于下限的节点；根据节点负载（分区数）动态迁移备分区以平衡

### 其他独立权（仅当存在多条独立权时填）
- **权 7（装置权利要求）**：与权 1 对应的方法权利要求，将步骤替换为"第一分配单元、第二分配单元、备分区移动单元"的装置实施例。F1-F4 以单元形式对应。

### 关键限定词与隐含约束
- 第一 / 第二 / 第三 / 第四预定阈值（K1, K2, 高负载阈值, 低负载阈值）— 均为正整数可配置参数
- 第二类节点 ≠ 第一类节点（主分区所在节点或所在分组）
- 备分区移动是高负载向低负载的单向迁移（F4 不覆盖反向）
- "多个主分区"和"多个节点"暗示 N≥2, M≥2
- "每个主分区对应于至少一个备分区" → 副本数 ≥2（1 主 + ≥1 备）
