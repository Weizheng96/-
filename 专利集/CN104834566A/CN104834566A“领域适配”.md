# CN104834566A 领域适配

## 领域分类

### 1. 主领域

- **主领域**：通信 / 云 / 网络（具体子方向：**虚拟交换 / NFV 数据面 / SDN 转发优化**）
- **次领域**：半导体 / 芯片设计（落入 DPU / SmartNIC 卸载场景时）

判定依据：专利权 1 / 9 / 17 显式限定"宿主机中的虚拟交换机 vSwitch"+"K 个转发处理线程"——核心场景是通用服务器宿主机内的 NFV 数据面调度；当 vSwitch fastpath 被卸载到 DPU / SmartNIC 时延伸到芯片侧。

### 2. 5 大主体类型清单（针对本领域的产业切片）

按"上游 → 中游 → 下游 → 用户 → 工具方"5 段切片：

| 编号 | 主体类型 | 角色 | 命中独立权 |
| --- | --- | --- | --- |
| **S1** | **vSwitch / SDN 软件 / 数据面软件提供方** | 直接 ship 转发处理线程调度逻辑的代码（开源项目 + 商业 fork + 商业控制器厂） | 权 9（装置） |
| **S2** | **NFV / 云服务 / 电信运营商部署方** | 在生产环境运行该 vSwitch 软件并触发动态调度 | 权 1（方法） |
| **S3** | **服务器整机 / DPU / SmartNIC 整机厂（OEM）** | 出厂产品中预装 vSwitch fastpath + 调度软件，作为"开箱即用 NFV 一体机" | 权 17（计算节点 / 整机） |

### 4. 适用规则集

| 标志 | 是否激活 | 判定依据 |
| --- | --- | --- |
| **R-OPENSOURCE** | ✅ **TRUE** | 本专利核心场景（vSwitch / DPDK / VPP / Cilium / OVS）几乎完全是开源生态——OVS / VPP / DPDK / Cilium 都是 Linux Foundation 项目；商业产品大量基于上游 fork。激活"上游开源项目本身列入 P0"+"商业发行版独立评估"+"上游贡献者归因检索"三条规则 |
| **R-CONFIG** | ✅ **TRUE** | F2-F5 命中往往体现为 PMD CPU 集合、`other_config:pmd-cpu-mask`、`pmd-rxq-affinity`、RSS Indirection Table 等具体配置参数；激活"config-driven 双引证"——必须同时出现 prose 描述 + 配置 key:value |
| **R-PARTNER** | ✅ **TRUE** | 本领域具有 vendor × 上游平台厂联合案例研究文化：Intel Network Builders、NVIDIA Partner / DOCA Showcase、Mellanox Reference Design 与 Red Hat / VMware / OpenStack 联合白皮书。激活第 6 类源（联合案例研究） |
| **R-PATENTWALL** | ✅ **TRUE** | 主分类 H04L 是公开度高的反向专利墙最有效领域之一；同时 G06F 9 / G06F 13 也常被 vSwitch / 调度类专利使用。激活反向专利墙两轮深读 |
| **R-PROCURE** | ✅ **TRUE** | 本专利的 S2（运营商 / 政企）场景下，集采招标书与技术应答书会公开声明 vSwitch / NFV 数据面性能要求。激活招标书检索 |
| **R-STANDARD** | ⚠️ **部分 TRUE** | 本专利不属于 ETSI / IEEE / 3GPP 强制标准必要专利（vSwitch 数据面调度不在 SEP 范围）；但**事实标准**（OVS / DPDK 是 NFV 行业事实参考实现）影响产业普及度。无 GB 强制国标。**判定**：R-STANDARD 不激活 Step 3c 标准合规子流程；但 ETSI NFV ISG / 3GPP TS 23.501 等 NFV 相关参考标准可作为 §A.5 行业标准源检索 |

### 5. 领域分类对后续步骤的影响

本专利领域分类为"通信 / 云 / 网络（NFV / 虚拟交换 / SDN 数据面调度）"。因此：

- **Step 3b 场景推演**：围绕 NFV 数据面调度展开 5+ 场景（云 vSwitch / 运营商 NFV / DPU 卸载 / 容器网络 / 边缘 NIC）
- **Step 4 候选清单**：按 S1-S5 5 大主体类型分组展开（每主体至少 3 家），重点点名 ODCC / FD.io / OPI 等行业协会成员；激活 R-OPENSOURCE 后**上游开源项目（OVS / DPDK / VPP）**与**商业发行版（Red Hat OpenStack / VMware NSX-T / Juniper Contrail）****双层独立成行**
- **Step 5 激活规则**：双引证（R-CONFIG）+ 联合案例研究（R-PARTNER）+ 反向专利墙两轮（R-PATENTWALL）+ 招标书源（R-PROCURE）+ 上游贡献者归因（R-OPENSOURCE）
- **Step 5 跳过规则**：标准合规判定子流程（R-STANDARD 不激活）
