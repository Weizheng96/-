# CN104834566A 潜在使用组织

> 收录闸门：仅 (i) 本领域知名组织 或 (ii) 估值≥10 亿美元未上市独角兽；且其产品须与 F1-Fk 同抽象层——即**自研 / 维护一个多核 vSwitch / 数据面转发引擎，存在"转发线程（PMD/worker/core）↔ 交换端口（port/rxq）"的运行时分配**。纯贴牌、纯下游集成、与转发线程调度不在同一层的厂商不列。

## 场景 1 / 2 / 3 共同的核心技术供给侧（vSwitch 多核数据面）

### A. 开源 / 标准实现的宿主组织（技术本体）

- **Open vSwitch（OVS）社区 / Linux Foundation**（类型 i 知名）— OVS 是事实标准虚拟交换机，`OVS-DPDK` 自带 `pmd-auto-lb`（PMD 线程级自动负载均衡 + RxQ-to-PMD 动态再分配），是 F1-F3 的教科书级实现 — 代表性已商用项目：Open vSwitch / OVS-DPDK（覆盖场景 1/2/3）
- **FD.io / VPP（Linux Foundation，思科发起）**（类型 i 知名）— 矢量包处理框架，worker 线程绑核 + `rx-placement` 把接口 rxq 分配到 worker，存在线程↔端口分配机制 — 代表性已商用项目：VPP（覆盖场景 1/2）

### B. 维护 / 商用化 vSwitch 数据面的知名厂商

- **Red Hat（IBM）**（类型 i 知名）— OVS-DPDK 主要上游维护者之一，`pmd-auto-lb` 多由其工程师贡献并写入 RHEL / OpenStack NFV 文档 — 代表性已商用产品：Red Hat OpenStack Platform（OVS-DPDK）、RHEL fast-datapath（场景 1/2）
- **VMware（Broadcom）**（类型 i 知名）— NSX / vSphere 的虚拟交换数据面（含 DPDK 加速 ESXi datapath、NSX-T edge），多核转发 — 代表性已商用产品：NSX-T Data Center、vSphere Distributed Switch（场景 1/2/3）
- **Cisco**（类型 i 知名）— VPP 发起方，企业级 vRouter / Cloud Services Router 数据面 — 代表性已商用产品：Cisco VPP-based 平台 / Catalyst SD-WAN vEdge（场景 1/3）
- **Juniper Networks**（类型 i 知名）— Contrail / Tungsten Fabric vRouter，DPDK 模式下转发核绑定 — 代表性已商用产品：Contrail Networking vRouter（场景 1/2）
- **Wind River**（类型 i 知名）— 电信级 NFVI（StarlingX 商用版），集成 OVS-DPDK — 代表性已商用产品：Wind River Studio / Titanium Cloud（场景 1）
