### 核心技术候选（依归并规则成立 — F1-F5 完整命中链路在该技术中已存在）

#### CT-1：**Open vSwitch PMD Auto Load Balance + pmd-rxq-rebalance + pmd-rxq-affinity**

- **核心技术**：OVS-DPDK 主线代码内置的"PMD Auto Load Balance"特性 + `ovs-appctl dpif-netdev/pmd-rxq-rebalance` 命令 + `pmd-rxq-affinity` 配置接口；自 OVS 2.9.0（2018）引入，长期 maintain
- **原创 / 权属组织**：
  - **Linux Foundation Networking** —— OVS 项目宿主
  - **OVS 主要 maintainer 公司**（commit history 排序）：**Red Hat**（多位核心 maintainer）、**Intel**、**Nicira / VMware**、**Ericsson**、**Cisco**、**Mellanox / NVIDIA**
- **类型**：开源项目 + 多 maintainer 共建
- **主体类型**：S1（vSwitch 软件提供方）—— 上游开源
- **关联的独立权**：权 1 / 权 9（共享 F1-F5）
- **产品族二级条目**（核心技术作为单条候选；其下游使用方融入证据材料）：
  - 核心技术本体：`upstream-ovs-pmd-autobalance`
- **命中场景**：场景 1（公有云）、场景 2（电信 NFV）、场景 4（容器网络中的 OVS-based CNI）
- **使用合理性**：OVS 是 NFV / 公有云 vSwitch 的事实标准；PMD Auto Load Balance 是 OVS 主线明文 feature；功能描述与权 1/9 F1-F5 高度对齐
- **可能性等级**：**高**
- **检索关键词建议**：`OVS PMD auto load balance`、`pmd-rxq-rebalance commit`、`other_config:pmd-auto-lb`、`pmd-rxq-affinity`、`OVS 2.9 release notes`、`ovs-vswitchd.conf.db pmd-load-threshold`

#### CT-2：**FD.io VPP NUMA-Aware Worker Thread Scheduler**

- **核心技术**：VPP 主线内置的 worker thread + NUMA-aware 端口绑定 + 动态 RSS 调整逻辑（`vlib`/`vnet` 调度框架）
- **原创 / 权属组织**：**FD.io / Linux Foundation Networking**（项目宿主）+ **Cisco**（VPP 主要贡献者，原 Cisco 开源给 FD.io）+ **Intel**
- **主体类型**：S1
- **关联的独立权**：权 1 / 权 9
- **可能性等级**：**中-高**（VPP 的"动态调度"力度低于 OVS 的 Auto Load Balance，但 NUMA 亲和性 + worker scheduler 路径与 F2/F3 有重叠）
- **检索关键词**：`VPP worker thread NUMA affinity`、`VPP RSS queue placement`、`vlib_worker`

| 候选名 | 名义所属领域 | 实际架构层级 | 与本专利 F1 的概念差距 | 排除依据 |
| --- | --- | --- | --- | --- |
| 网宿科技 ChinaNetCenter | 云 / CDN | CDN PoP 节点 — 流量调度在四层负载均衡器层 | 不在宿主机 vSwitch 转发线程层；F1 不命中 | 架构层级不符 |
| Cloudflare 边缘 | 边缘云 / DDoS 防护 | 边缘 PoP — 自有 user-space TCP/UDP stack（quiche、Pingora），不使用 vSwitch | F1 不命中（Pingora 是应用层服务，非虚拟交换） | 架构层级不符 |
| 各类 SaaS 应用层 LB / Service Mesh（Istio / Linkerd / Envoy） | 微服务网络 | Sidecar / mesh 代理 — 用户应用层 | F1 不命中（不在宿主机虚拟交换层）| 架构层级不符 |
| Kernel 内置 IRQ balancing / cgroup | 通用 OS | OS 调度器 — 调整 CPU pin 而非端口 | F5 不命中（调整线程的 CPU 而非"线程所服务的端口"） | 架构层级不符 |

预排除候选**保留在 Step 6 状态总表"第 5 档：已排除（架构层级不符）"**作为审计可追溯条目。

## 领域过拟合自检

本 Step 4 候选清单**没有**包含任何与本专利领域（通信 / 云 / 网络 NFV / 虚拟交换数据面调度）无关的厂商示例。所有候选都直接对应 vSwitch / NFV / DPU / 容器网络数据面调度场景；没有把医疗 / 金融 / 工业自动化 / 生物制药 / 等无关领域的厂商误纳入。
