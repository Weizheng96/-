# CN107678790B 潜在使用组织

> **场景合并说明**：Step 2 的三个背景场景（金融服务 / 传感监测 / 网络监控）在抽象层上共用**同一类潜在侵权产品**——"支持以 SQL 定义流作业、并把 SQL 编译成逻辑流图→物理算子 DAG（含算子合并 / operator chaining）下发到 master-worker 集群执行的流计算引擎 / 平台"。真正实践 F1–F5 的是这类**引擎 / 平台的提供方**，而非单纯跑业务的下游金融 / IoT / 网安公司（后者只是引擎的用户，不自建该机制）。因此本步按"引擎 / 平台提供方"统一枚举，并标注其覆盖的场景（绝大多数同时覆盖 1/2/3 三场景，因为流计算引擎是通用的）。
>
> **闸门**：仅收录 (i) 知名组织 或 (ii) 独角兽（≥10 亿美元未上市）；且其产品须**自建** SQL→流图→算子 DAG 的流计算引擎（同抽象层）。纯消息中间件 / Broker（如 Kafka/Pulsar/Redpanda 本体）、纯存储查询（Pulsar SQL）、纯 OLAP 物化视图（与流图算子调度不在同一层）一律不列。
>
> **权利人自身排除**：华为（华为云 DLI / CS 流计算）是本专利权利人，不作为侵权候选。
