   → 命中多为"小窗模式"（floating window，边角内滑呼出），与"分屏"是两套功能；需收窄到"分屏"。
2. `魅族 Flyme 分屏 上下分屏 调换 交换 两个窗口 位置 拖动`
   → 关键信号：分屏进入 = 屏幕下方两侧上滑→卡片式多任务→点击卡片右上角"分屏"按钮（菜单/按钮，非滑动划线分屏）；中间分界线可拖动调整大小；**未找到上下两窗口内容互换操作**。
3. `魅族 Flyme 分屏 滑动手势 进入 横向 纵向 划线 三指`
   → 分屏自 Flyme 5 起；触发为多任务界面操作；小窗才是"屏幕边缘竖直方向内滑呼出"。
4. `Flyme 分屏 上下两个应用 调换位置 拖到顶部 交换 swap split screen`
   → 无任何"分屏窗口内容互换 / swap"证据。
5. `Flyme 分屏 左右分屏 横屏 调换上下应用 拖动分隔条 交换内容`（深抓阶段补充）
   → 分屏自 Flyme 5；分界线可拖动调大小；无方向映射、无内容互换证据。

## 关键结论
- 分屏触发 = 多任务卡片 + 下滑菜单【分屏】按钮（非"沿触摸屏横向/纵向滑动"划线分屏）。
- 分屏方向 = 固定上下排列；分界线仅可拖动调大小；无"横向滑动→上下、纵向滑动→左右"方向映射。
- 两个分屏窗口内容互换（F3+F4，本专利最具区分性特征）= 全部来源均未见 → 公开资料不足（非正向反据）。
- 注：Flyme"小窗模式"才用屏幕边缘内滑手势呼出，但小窗≠分屏，且小窗是呼出悬浮窗、非按滑动方向把屏幕一分为二，亦无双窗内容互换。
