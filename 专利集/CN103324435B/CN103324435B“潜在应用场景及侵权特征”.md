## 一、潜在应用场景（严格来自背景技术 — 不外扩）

### 场景 1：移动智能终端的触摸屏分屏多任务
- **背景技术 verbatim 依据**：
  > "[0002] 当前智能手机、平板电脑等移动终端设备日渐普及，这些设备大都采用大屏幕，通过触摸屏幕进行操作。"
  > "[0003] 现有技术中，终端的后台可以启动多个程序，但屏幕只能显示一个应用程序，若需显示另外一个应用程序则需要返回当前程序的应用程序，从而对用户操作带来不便。"
- **场景说明**：智能手机、平板电脑等带触摸屏的移动终端的终端用户，在需要同时查看 / 操作多个应用程序时，希望不在程序间频繁切换，而是把屏幕分成多个窗口并行显示，并能在窗口之间调整 / 交换显示内容。
- **使用本专利的典型形式**：移动终端操作系统 / 系统 UI 中内置的"分屏 / 多窗口"特性——以触摸屏滑动手势触发屏幕区域划分（横向滑动→上下分屏、纵向滑动→左右分屏），并以窗口间滑动手势实现两个分屏窗口的显示内容互换。（仅基于背景技术明示的"移动终端 + 触摸屏 + 多程序显示"，不外扩到桌面 PC / 车机 / 电视等背景未提及的形态。）

> 背景技术只描述了这一个场景（移动终端触摸屏多任务显示），故只列 1 个场景——宁少勿编。

## 二、独立权利要求 + 侵权特征 F1-Fk

本专利有 3 条独立权利要求：权 1（方法）、权 24（装置）、权 47（电子设备）。三者技术内核完全一致（获取滑动信号→分屏→窗口间滑动→互换内容），仅表达载体不同。F1-F4 以方法权 1 为锚，权 24 / 权 47 同理命中。

### 权 1 verbatim
> 1. 一种分屏显示的方法，其特征在于，所述方法应用于具有触摸屏的电子设备，所述方法包括：
> 获取通过所述触摸屏输入的分屏触摸信号，所述分屏触摸信号包括沿所述触摸屏的横向滑动信号或者沿所述触摸屏的纵向滑动信号；
> 根据所述横向滑动信号，将所述触摸屏的当前显示区域划分为上下排列的至少两个显示窗口；或者，根据所述纵向滑动信号，将所述触摸屏的当前显示区域划分为左右排列的至少两个显示窗口；
> 获取第一显示窗口至第二显示窗口的滑动触摸信号或者所述第二显示窗口至所述第一显示窗口的滑动触摸信号；
> 根据所述滑动触摸信号将所述第一显示窗口内的显示内容切换至所述第二显示窗口内显示，并将所述第二显示窗口内的显示内容切换至所述第一显示窗口内显示。

### F1-F4 清单

- **F1（分屏触摸信号 = 触摸屏滑动手势）｜[OR]**：> "获取通过所述触摸屏输入的分屏触摸信号，所述分屏触摸信号包括沿所述触摸屏的横向滑动信号或者沿所述触摸屏的纵向滑动信号"
  - 可能描述方式：① 在触摸屏上用手指做一次横向（水平）或纵向（竖直）滑动 / 划线手势触发分屏；② "swipe to split" / "用滑动手势进入分屏"；③ 三指 / 单指横扫 / 竖扫进入分屏；④ 在屏幕中部画一条分隔线分屏。（[OR]：横向滑动信号 **或** 纵向滑动信号，命中任一即满足 F1。）

- **F2（按滑动方向把当前显示区域划分为至少两个显示窗口）｜[OR]**：> "根据所述横向滑动信号，将所述触摸屏的当前显示区域划分为上下排列的至少两个显示窗口；或者，根据所述纵向滑动信号，将所述触摸屏的当前显示区域划分为左右排列的至少两个显示窗口"
  - 可能描述方式：① 横向滑动→上下两个窗口（上下分屏）；纵向滑动→左右两个窗口（左右分屏）；② "split screen into top/bottom (or left/right) windows"；③ 进入双窗口 / 多窗口（multi-window）并排显示；④ 当前前台应用所在区域被一分为二，各占一个窗口。（[OR]：横向→上下 **或** 纵向→左右，命中任一分支即满足 F2。隐含整数限定见下方"关键限定词"——"至少两个"即 N≥2。）

- **F3（获取两个显示窗口之间的滑动触摸信号）｜[AND]**：> "获取第一显示窗口至第二显示窗口的滑动触摸信号或者所述第二显示窗口至所述第一显示窗口的滑动触摸信号"
  - 可能描述方式：① 从一个分屏窗口向另一个分屏窗口滑动 / 拖动的跨窗口手势；② "drag from window A to window B" / 拖拽跨越窗口边界；③ 长按某窗口后拖到另一窗口；④ 跨分屏区域的滑动 / 拖放交互。（"第一→第二 或 第二→第一"指方向任一，本特征核心是"存在窗口间的滑动触摸信号"。）

- **F4（依该滑动信号互换两个窗口的显示内容）｜[AND]**：> "根据所述滑动触摸信号将所述第一显示窗口内的显示内容切换至所述第二显示窗口内显示，并将所述第二显示窗口内的显示内容切换至所述第一显示窗口内显示"
  - 可能描述方式：① 把上 / 左窗口的应用与下 / 右窗口的应用对调位置（两个窗口内容互换 / swap）；② "swap the two split apps / panes"、"交换分屏窗口"；③ 拖放后两个窗口的应用程序位置互换；④ A、B 两窗口内容双向对调（A 的内容到 B、B 的内容到 A）。（[AND]：必须是**双向互换**——既把第一窗口内容换到第二，又把第二窗口内容换到第一；仅把一个窗口替换为新应用、而另一窗口不变，不落入 F4。）

**结构标签规则说明**：F1、F2 为 [OR]（候选命中任一分支即满足）；F3、F4 为该方法的必经步骤，F4 内部"两个方向都换"为 [AND]（双向互换缺一不可）。

### 其他独立权（存在多条独立权）
- 权 24（装置）：信号获取模块 + 分屏显示模块 + 显示内容切换模块，功能与权 1 的 F1-F4 一一对应——装置实施 F1-F4 即落入。
- 权 47（电子设备）：存储器 + 处理器 + 触摸屏；触摸屏负责获取分屏触摸信号 / 窗口间滑动触摸信号，处理器负责分屏划分与内容互换——电子设备实施 F1-F4 即落入。

### 关键限定词与隐含约束
- **触摸屏载体**：方法 / 装置 / 设备均"应用于具有触摸屏的电子设备"——纯物理按键 / 鼠标键盘、无触摸屏的设备不落入（同抽象层判断点）。
- **整数限定 N≥2**：分屏后为"至少两个显示窗口"——单窗口不算；F4 的内容互换针对"第一 / 第二"两个窗口（≥2 个窗口中的某两个）。
- **触发方式 = 滑动手势**：分屏由"沿触摸屏横向 / 纵向的滑动信号"触发，而非菜单 / 按钮 / 快捷键 / 系统推荐。若候选仅通过菜单 / 多任务卡片按钮进入分屏、无滑动手势触发分屏，则 F1 需个案判断是否等同。
- **方向映射**：横向滑动↔上下排列；纵向滑动↔左右排列（F2 的两个分支各自对应）。
- **内容互换 vs 替换**：F3+F4 的核心区分性特征是"以窗口间滑动手势把两个已分屏窗口的内容**互换**"，区别于"在某个空窗口里新开一个应用"。这是本专利相对一般分屏的最具区分性特征——拆 F# 与 Step 6 取证时重点核查候选是否具备"两个分屏窗口内容互换 / 对调"交互。
