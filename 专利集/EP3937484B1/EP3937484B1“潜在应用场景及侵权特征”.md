- 标题：CONTENT-AWARE ENERGY SAVINGS FOR WEB BROWSING UTILIZING SELECTIVE LOADING PRIORITY（基于内容感知、利用选择性加载优先级的网页浏览节能）

## 一、潜在应用场景（严格来自背景技术 — 不外扩）

### 场景 1：移动设备上的网页浏览应用（浏览器 / 应用内渲染引擎）省电
- **背景技术 verbatim 依据**：
  > "[0003] ... one of the typical applications being used on these devices is a web browsing application. These applications request markup language documents from a server, parse the markup language document, and then render the elements included in the markup language document to a window displayed by the mobile device. Power is consumed by downloading large files ... rendering text, images, or video data to produce pixel data for display; and updating the display ..."
  > "[0002] The battery capacity of mobile devices such as a tablet computers and smart phones is extremely important ... software solutions have been implemented to reduce the power consumption ... such as by reducing the number of active threads in a low-power mode or reducing the brightness of a display."
- **场景说明**：智能手机 / 平板上的网页浏览应用（独立浏览器，或操作系统/厂商内置浏览器，或 App 内嵌的 WebView/渲染引擎），在请求并渲染 HTML 等标记语言文档时消耗大量电量；本专利提出在浏览/渲染层按"元素优先级 + 错峰延迟显示"省电。
- **使用本专利的典型形式**（仅基于背景技术 + SUMMARY "a window of an application" 明示，不外扩）：
  - 移动端独立浏览器 App 的渲染/加载调度模块
  - 操作系统或设备厂商内置浏览器 / 系统级 WebView 渲染引擎
  - App 内嵌 WebView / 浏览器内核，用于渲染网页或 H5 信息流页面

> 背景技术仅实质描述了"移动网页浏览"这一个场景，故只列 1 个场景（宁少勿编）。专利的发明点在于"浏览/渲染引擎层的元素优先级 + 延迟显示省电机制"，与具体业务（新闻、电商、社交等）无关——业务 App 只有在其自有渲染/加载调度实现了该机制时才落入。

## 二、独立权利要求 + 侵权特征 F1-Fk

### 权 1 verbatim
> 1. A computer-implemented method (700) for web browsing on a mobile device, comprising:
> - receiving (702) a markup language document that includes a plurality of elements;
> - assigning (704) the plurality of elements a priority value based on a set of rules, wherein the set of rules includes at least two rules configured to assign two or more priority values to at least two subsets of elements, in the plurality of elements, wherein the subsets of elements comprise a parent element and its nested elements;
> - determining (752) an order of elements based on the priority value assigned to the plurality of elements; and
> - displaying (754), in the order of elements, rendered content corresponding to each element in the plurality of elements, wherein the display of a next element in the order of elements is delayed for a predetermined delay time after displaying a previous element in the order of elements.

### F1-F7 清单

权 1 是方法权，各步骤为顺序的必要限定，整体按 [AND] 组合（候选须全部命中才落入权 1）。下方对每个 F# 标 [OR]/[AND] 指其**内部子结构**。

- **F1（接收含多元素的标记语言文档）｜[AND]**：> "receiving (702) a markup language document that includes a plurality of elements"
  - 可能描述方式：浏览器/WebView 请求并解析 HTML/XML/标记文档；加载页面构建 DOM 树（含多个元素节点）；"fetch HTML and parse into a document with multiple elements"；解析 markup 得到 element 列表。

- **F2（基于规则集给元素分配优先级值）｜[AND]**：> "assigning (704) the plurality of elements a priority value based on a set of rules"
  - 可能描述方式：为页面元素/资源打优先级标签（如 high/low/lazy、`fetchpriority`、`loading=lazy`）；基于规则引擎/启发式给元素赋权重或分级；priority hints / resource scheduling 给 DOM 节点或资源排优先级。

- **F3（规则集含 ≥2 条规则、产生 ≥2 个优先级值、作用于 ≥2 个元素子集）｜[AND]（含整数限定）**：> "the set of rules includes at least two rules configured to assign two or more priority values to at least two subsets of elements"
  - 可能描述方式：至少两类规则共同决定优先级（如"按元素类型"+"按是否广告"/"按是否视口内"+"按资源大小"）；产生 ≥2 档优先级（如高/低，或 high/medium/low）；对至少两组元素子集分别赋不同优先级。
  - ⚠️ 整数限定：须有 ≥2 条规则 且 ≥2 个优先级档 且 ≥2 个子集。若候选只有单一规则或仅"二值在/不在视口"的一条规则，需审视是否满足"至少两条规则"。

- **F4（元素子集包含"父元素及其嵌套元素"——基于 DOM 树父子结构定优先级）｜[AND]（结构限定）**：> "wherein the subsets of elements comprise a parent element and its nested elements"
  - 可能描述方式：按 DOM 树/嵌套结构组织优先级，容器（父）元素与其子（嵌套）元素作为同一优先级子集/子树一起处理；subtree/container-based prioritization；对一个区块（含其内部子节点）整体赋优先级。

- **F5（基于优先级值确定元素显示顺序）｜[AND]**：> "determining (752) an order of elements based on the priority value assigned to the plurality of elements"
  - 可能描述方式：按优先级对渲染/加载队列排序；reorder loading/paint order；调度器按 priority 出队决定先后；高优先元素先、低优先后。

- **F6（按该顺序显示各元素对应的渲染内容）｜[AND]**：> "displaying (754), in the order of elements, rendered content corresponding to each element in the plurality of elements"
  - 可能描述方式：按确定的顺序逐元素渲染/绘制到窗口；progressive rendering，按优先级先后把内容贴到屏幕。

- **F7（下一元素的显示相对上一元素延迟一个"预定延迟时间"——错峰/分批延迟显示）｜[AND]（关键区分特征 + 时间限定）**：> "wherein the display of a next element in the order of elements is delayed for a predetermined delay time after displaying a previous element in the order of elements"
  - 可能描述方式：相邻元素的显示之间插入一个**预先设定的延迟时长**；分批/错峰渲染，批与批（或元素与元素）之间定时间隔；"stagger / throttle rendering with a fixed inter-element delay"；以定时器分时把元素逐个上屏（time-sliced display with a preset gap）。
  - ⚠️ 这是相对 prior art（US2012/144288 仅按带宽分配渲染；US2013/227398 仅把页面拆成多个离散屏页逐屏显示）的**最关键区分特征**：必须是"相邻元素显示之间有一个**预定的延迟时间**"，而非单纯的异步加载/懒加载（懒加载是"进入视口才加载"，不必然是"上一个显示后等待固定时长再显示下一个"）。取证时须区分：是否有**确定的、元素间的延迟时长**机制。

### 其他独立权（与权 1 内核一致，仅形式不同）
- **权 7（移动设备/装置）**：权 1 的"存储指令的存储器 + 一个或多个处理器执行"装置镜像，限定与 F1-F7 等同。
- **权 12 / 权 13（非暂态计算机可读介质/存储介质）**：存储指令、执行时实现权 1-6 的方法（Beauregard 型），内核同权 1。

> 因四条独权技术内核同一，Step 6 取证以 F1-F7 为统一靶点；候选只要实现权 1 的方法即可同时落入权 7（装置）与权 12/13（介质）。

### 关键限定词与隐含约束
- **整数限定（F3）**：set of rules ≥2 条规则；≥2 个（"two or more"）优先级值；≥2 个（"at least two"）元素子集。
- **结构限定（F4）**：优先级子集须是"父元素 + 其嵌套元素"——即优先级须挂在 DOM 树的父子/嵌套结构上，而非扁平的资源列表。
- **时间限定（F7）**：相邻元素显示之间须有一个 **predetermined delay time**（预定延迟时间）——是"上一个显示后、延迟固定时长再显示下一个"，区别于普通异步/懒加载/优先级提示。
- **设备限定**：mobile device（智能手机 / 平板等移动设备）。
- **省电语境**：整套机制服务于降低移动设备 web 浏览功耗（SUMMARY/背景），但权 1 字面未把"实测省电"作为限定步骤——命中以 F1-F7 字面/等同为准。
- **依赖权附加特征（仅作 Step 6 加分项，非权 1 必要项）**：权 2 按元素分类赋优先级；权 3 按"是否广告内容"赋优先级；权 4 规则集按 URL 选取；权 5 normal/low-power 双模式按优先级范围加载（[OR] 双分支）；权 6 低功耗模式手段 = DVFS / 降分辨率 / 降亮度 / 限制低优先级元素线程 / 遮蔽低优先级元素 中至少一种（[OR] 多分支）。
