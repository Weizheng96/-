# EP3937484B1 潜在使用组织

> 场景仅 1 个（移动设备网页浏览/渲染引擎省电）。下列主体均通过两道闸：(i) 知名组织 或 (ii) 独角兽；且其产品在概念上落在 F1-F7 同一抽象层 —— 即拥有**自研或深度定制的移动端网页渲染/加载调度引擎**，有可能自行实现"元素优先级 + 按序错峰延迟显示"。纯下游集成、不碰渲染调度的应用一律不列。

### 场景 1：移动设备上的网页浏览应用（浏览器 / 应用内渲染引擎）省电

**A. 浏览器内核 / 引擎自研方（最贴 F1-F7 抽象层）**

- **Google**（知名 / i）— 全球最大浏览器内核 Blink/Chromium 的主导方；其移动端有 Chrome (Android) 与 Android System WebView **两条独立产品线**，均自研渲染/资源调度器（priority hints、lazy-loading、Lite/Data Saver 模式）— 代表性已商用产品：Chrome for Android, Android System WebView, Chrome "Lite mode/Data Saver"。
- **Apple**（知名 / i）— WebKit 引擎与 iOS 唯一允许的底层渲染内核；自研渲染调度、以省电著称 — 代表性产品：Safari (iOS), WKWebView。
- **Mozilla**（知名 / i）— Gecko 引擎；移动端 Firefox 与 GeckoView 自研资源/渲染调度 — 代表性产品：Firefox for Android, GeckoView。
- **Microsoft**（知名 / i）— Edge（Chromium 内核）移动版，叠加自有效率/省电特性 — 代表性产品：Microsoft Edge (Android/iOS)。

**B. 自研/深度定制内核的移动浏览器厂商**

- **Opera Limited**（知名 / i；NASDAQ: OPRA）— 数据节省浏览器的标杆，Opera Mini 压缩 + 电池保护模式 — 代表性产品：Opera for Android, Opera Mini（注：Mini 极速模式为服务端渲染，端侧是否做元素级延迟显示需 Step 6 核）。
- **Alibaba / UCWeb**（知名 / i）— UC 浏览器 U4 内核，主打压缩 + 智能/分级加载 — 代表性产品：UC Browser（U4 kernel）, UC Browser Mini。
- **Tencent**（知名 / i）— QQ 浏览器 X5/TBS 内核；该内核同时被微信内置浏览器（WebView）复用，**跨"独立浏览器 + 超级 App 内嵌 WebView"两条产品线**，均自研渲染调度 — 代表性产品：QQ Browser (X5 kernel), 微信内置浏览器/TBS。
- **Baidu**（知名 / i）— 百度浏览器及百度 App 内嵌浏览内核 — 代表性产品：百度浏览器, 百度 App 内置浏览器。
- **Qihoo 360**（知名 / i）— 360 手机浏览器 — 代表性产品：360 浏览器（移动版）。

**C. 设备厂商内置浏览器（OEM，均为大型/上市公司，自有省电/数据节省模块）**

- **Samsung Electronics**（知名 / i）— Samsung Internet（Chromium 基础上叠加自有高对比/省电/数据节省）— 代表性产品：Samsung Internet Browser。
- **Xiaomi**（知名 / i）— 小米浏览器 / Mint Browser — 代表性产品：Mi Browser, Mint Browser。
- **vivo**（知名 / i）— vivo 浏览器 — 代表性产品：vivo 浏览器。
- **OPPO / Heytap**（知名 / i）— Heytap/OPPO 浏览器 — 代表性产品：Heytap Browser。

**D. 隐私/数据节省细分领域知名玩家**

- **Brave Software**（知名 / i；隐私浏览器领域公认头部）— Brave（Chromium 内核 + 自有省电/省流/拦截层）— 代表性产品：Brave Browser (Android/iOS)。
- **CloudMosa**（数据节省细分知名 / i）— Puffin Browser，云端渲染（端侧主要显示云端渲染流，是否在端做元素级延迟显示存疑，Step 6 核）— 代表性产品：Puffin Browser。
- **Aloha Mobile**（隐私浏览器细分较知名 / i）— Aloha Browser，宣称自研 "Aloha Core" 引擎 — 代表性产品：Aloha Browser。

**Step 3 完备性自检**：
① 单一场景已对"自研/定制移动渲染引擎且做加载/渲染调度"的玩家做枚举（A 引擎方 / B 自研内核浏览器 / C OEM 内置 / D 细分头部），非仅举代表。
② 跨多产品线集团已分别补列：Google（Chrome + WebView + Lite mode）、Tencent（QQ Browser + 微信内置 X5/TBS）、Alibaba/UCWeb（UC + UC Mini）。
③ 反向核查：均为"做这件事"（自有移动 web 渲染/加载调度）的主体；纯应用层、不碰渲染内核的 App（仅调用系统 WebView 默认行为者）未列入，避免无关层。
④ 专利权人 Huawei 自身（及其浏览器）不作为侵权候选，排除。
⑤ 云渲染代理型（Opera Mini 极速模式 / Puffin）已标注"端侧是否做元素级延迟显示存疑"，留待 Step 6 按同抽象层硬条件甄别。

> 说明：本场景属"系统/内核级机制"，真正可能实现 F1-F7（尤其 F4 父子嵌套优先级 + F7 元素间预定延迟）的是**自研或深度定制渲染调度的主体**。仅调用上游 Chromium/WebKit 默认行为、未自研调度的下游 App 不在同抽象层，未列。
