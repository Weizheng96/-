## 4. 适用规则集

| 标志 | 是否激活 | 判定依据 |
| --- | --- | --- |
| **R-OPENSOURCE** | ⚠️ 部分 | TEE / Merkle 树 / 区块链相关有大量开源（Ethereum / Hyperledger / OP-TEE / TrustZone / Apache Tuweni 等） |
| **R-CONFIG** | ⚠️ 部分 | 涉及配置：私钥生成方式、TEE 配置、Merkle 分支因子、签名算法选择 |
| **R-PARTNER** | ✅ TRUE | 车厂 × 车联网安全方案厂联合白皮书丰富（特别是与区块链方案厂的合作）|
| **R-PATENTWALL** | ✅ TRUE | G06F21 / H04L9 是高公开度领域 |
| **R-PROCURE** | ✅ TRUE | 政企监管平台 + 整车厂多通过招标采购车联网安全方案 |
| **R-STANDARD** | ⚠️ 部分 | GB/T 40855《电动汽车远程服务与管理系统通信协议》、GB/T 40861《汽车信息安全通用技术要求》、ISO/SAE 21434《道路车辆 — 网络安全工程》、UN R155/R156（联合国汽车网络安全法规）— 这些是行业准入标准；但本专利涉及"双节点 + Merkle 树"具体优化，未必构成 SEP；R-STANDARD 不激活 Step 3c 子流程 |
