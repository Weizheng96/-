## 入选说明

**为什么以下未列入**（Step 6 资源保护）：
- **纯芯片厂**（地平线 / 寒武纪 / Mobileye EyeQ 芯片本身）：在抽象层上是 SoC + BPU，不直接实现"采集 → Merkle 根 → 签名包头 → 上传"完整链路，由 OEM 在芯片之上的 OS / 中间件做。Mobileye REM 是 Mobileye 自己实现的上传链路，所以单独列了 07。
- **纯地图厂**（四维图新 / 中海庭）：抓的是地图数据 + 众包采集，与"行车事故证据"目标不一致；F2 命中弱。
- **保险公司 UBI**（平安 / 人保）：他们消费数据而非生成数据，理论上可作辅助节点但目前公开方案中没有"包头同步 + 验证"的对应实现。
- **京东智臻链 / 各通用司法链 SDK**：未明确部署到车联网行车数据场景；待 Step 5 长尾扩展时如有真实部署再补。
- **特斯拉 Autopilot 模型推理本身**：抽象层是 ML 模型，与本专利的数据签名机制不同；只列其 EDR 子系统（01）。
