# 检索留痕 — 15-tencent-smartnic（腾讯云自研智能网卡云网络卸载平台）

公开日时间窗：仅采信 **2023-06-06（专利授权公告日）之后** 的实质性公开；下列文章发布日期一并记录用于时间核验。

## Phase 1 粗筛（WebSearch，串行）

1. `腾讯云 智能网卡 DPU 银杉 OR 水滴 虚拟交换 流表卸载`
   - 命中：腾讯自研 DPU 水杉（2020 上线，2x50G）、银杉（2021 上线，2x100G）做云网络/云盘数据面硬件卸载、vSwitch 卸载。**有信号**（确为同抽象层产品）。
2. `腾讯云 智能网卡 多网卡 链路聚合 LACP 流表 卸载 高可靠 单点故障`
   - 命中：均为腾讯云开发者社区的**通用 LACP/bond 科普文**，非腾讯 DPU 自身架构；与本专利"多网卡流表同步卸载"无直接对应。弱信号。
3. `腾讯 自研DPU 水杉 银杉 双网卡 bond 高可用 网卡故障 vswitch 卸载 架构`
   - 命中：通用 bond 教程 + 火山引擎 vSwitch（他厂）；腾讯侧仅泛泛"自研 DPU 卸载全量规则"，无多卡聚合细节。弱信号。

## Phase 2 深抓（WebFetch，串行）

4. https://cloud.tencent.com.cn/developer/article/2105157 （腾讯自研DPU技术演进，发布 **2022-09-14**）
   - DPU = **单卡双端口**（2x100G 在一块卡上），每 Socket 直通 PCIe 到 DPU；
   - 容错为 **Harp 协议多路径**（网络路径级，非网卡硬件级多卡冗余）；
   - **未提及** 跨多块独立网卡聚合 / LACP / 流表向多卡同步 / 网卡级单点故障方案。
5. https://cloud.tencent.com/developer/article/2106619 （行稳致远：腾讯自研DPU商用情况与技术演进，发布 **2022-09-14**）
   - 高可用靠 **热迁移**（自研 DPU 卡虚机 ↔ 标卡虚机互迁）+ **Harp 多路径**；
   - **未提及** LACP 多卡聚合、流表多卡同步、miss 触发下发。
6. https://cloud.tencent.cn/developer/article/2191553 （软硬件融合技术内幕，发布 **2022-12-13**）
   - 仅"物理网卡内部集成 vSwitch、具备 vSwitch 卸载功能"泛述；
   - 多网卡流表同步 / LACP / miss 触发 / 单网卡单点故障多卡方案 **均未涉及**。

## 留痕小结
- 腾讯自研 DPU 确为本专利同抽象层产品（vSwitch 流表硬件卸载），但 3 篇权威技术文章一致显示其为**单卡双端口架构**，冗余/高可用走**热迁移 + Harp 多路径**这一**不同机制**，而非本专利"N≥2 独立网卡 LACP 聚合为第一端口 + 精确流表 miss 触发同步卸载至全部 N 卡"。
- 受限说明：腾讯 DPU 网卡级细节公开度低；现有 3 篇为截至检索可得的最权威公开来源，均无 F2/F3/F5 正向证据，F1（N≥2 独立网卡）亦无支撑——其叙述的是单卡 + 路径级/迁移级容错，构成"不同机制"反向信号。
