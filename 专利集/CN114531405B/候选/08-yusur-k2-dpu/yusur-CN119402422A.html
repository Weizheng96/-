Abstract

Translated from Chinese

本公开涉及一种DPU网卡硬件流表卸载方法、装置、设备及介质。其中，DPU网卡硬件流表卸载方法包括：接收用户的运维工具下发的BPF字节码；根据所述BPF字节码更新DPU网卡的硬件流表组件；通过所述硬件流表组件对接收到的网络数据流进行路径匹配处理，得到对应的目标路径；基于所述目标路径对所述网络数据流进行传输。根据本公开实施例，能够降低适配成本，减少用户操作，提升用户体验。

The present disclosure relates to a method, device, equipment and medium for unloading hardware flow tables of a DPU network card. The method for unloading hardware flow tables of a DPU network card includes: receiving the BPF bytecode sent by the user's operation and maintenance tool; updating the hardware flow table component of the DPU network card according to the BPF bytecode; performing path matching processing on the received network data flow through the hardware flow table component to obtain the corresponding target path; and transmitting the network data flow based on the target path. According to the embodiments of the present disclosure, it is possible to reduce adaptation costs, reduce user operations, and improve user experience.

Description

DPU network card hardware flow table unloading method, device, equipment and medium
Technical Field
The disclosure relates to the technical field of intelligent acceleration DPU network cards, and in particular relates to a DPU network card hardware flow table unloading method, device, equipment and medium.
Background
When the existing DPU network card is unloaded in a flow table and then a specific five-tuple data message is sent to another quick path, the used tool is a specific tool, and the interface is not standard. The user needs to change the operation tool of the user when using the DPU network card of the equipment manufacturer to support the function of the DPU network card.
Therefore, when users use the DPU network cards of different equipment vendors, different operation and maintenance tools are needed to support the DPU network cards of different equipment vendors, the adaptation cost is high, and the user operation is complex.
Disclosure of Invention
In order to solve the technical problems, the present disclosure provides a method, a device, equipment and a medium for unloading a hardware flow table of a DPU network card.
In a first aspect, the present disclosure provides a method for unloading a hardware flow table of a DPU network card, including:
Receiving BPF byte codes issued by an operation and maintenance tool of a user;
Updating a hardware flow table component of the DPU network card according to the BPF byte code;
Performing path matching processing on the received network data flow through the hardware flow table component to obtain a corresponding target path;
and transmitting the network data stream based on the target path.
In a second aspect, the present disclosure provides a DPU network card hardware flow table unloading apparatus, including:
the data receiving module is used for receiving BPF byte codes issued by the operation and maintenance tool of the user;
The flow table updating module is used for updating a hardware flow table component of the DPU network card according to the BPF byte code;
the path matching module is used for carrying out path matching processing on the received network data flow through the hardware flow table component to obtain a corresponding target path;
and the data transmission module is used for transmitting the network data stream based on the target path.
In a third aspect, the present disclosure provides a DPU network card hardware flow table unloading device, including:
A processor;
a memory for storing executable instructions;
The processor is configured to read the executable instructions from the memory, and execute the executable instructions to implement the DPU network card hardware flow table unloading method of the first aspect.
In a fourth aspect, the present disclosure provides a computer readable storage medium storing a computer program, which when executed by a processor, causes the processor to implement the DPU network card hardware flow table offloading method of the first aspect.
Compared with the prior art, the technical scheme provided by the embodiment of the disclosure has the following advantages:
according to the DPU network card hardware flow table unloading method, device, equipment and medium, BPF byte codes issued by operation and maintenance tools of users can be received, then the hardware flow table components of the DPU network card are updated according to the BPF byte codes, then path matching processing is carried out on received network data flows through the hardware flow table components to obtain corresponding target paths, finally the network data flows are transmitted based on the target paths, therefore, the hardware flow table components of the DPU network card can be updated according to the BPF byte codes issued by the operation and maintenance tools of the users, the corresponding target paths are determined to be transmitted through the hardware flow table components after the network data flows are received, and therefore the hardware flow table components of the DPU network card are updated according to the operation and maintenance tools of the users without changing the operation and maintenance tools of the users, the adaptation cost is reduced, the user operation is improved, and the user experience is improved.

Optionally, the data processing unit or dedicated data processor (Data Processing Unit, DPU) is a hardware accelerator for data processing running a separate operating system, complex hardware offload modules and network, security, storage software functions matched thereto. DPUs are commonly used to accelerate throughput-intensive tasks in data centers, cloud computing, and edge computing, such as artificial intelligence reasoning, large-scale data analysis, and network security. The DPU consists of three main elements including a processing core, a network interface, and a memory interface. These components work together to enable the DPU to efficiently process data.

In the embodiment of the disclosure, the DPU network card hardware flow table unloading method can be executed by the electronic device. Wherein the electronic device may include, but is not limited to, a device such as a computer device, cloud server, or cloud server cluster. The electronic device may include a DPU network card.
As shown in fig. 1, the DPU network card hardware flow table unloading method may include the following steps.
S110, receiving BPF byte codes issued by an operation and maintenance tool of a user.
In the embodiment of the disclosure, the electronic device may receive the BPF bytecode issued by the user's operation and maintenance tool.
Optionally, the operation and maintenance tool refers to a tool that software performs work such as server configuration, deployment, management, monitoring and the like instead of manual operation.
Optionally, berkeley Packet Filter (BPF) bytecode is a binary intermediate code issued by the user's operation and maintenance tool.
Specifically, the operation and maintenance tool of the user software can send the BPF byte code to the DPU network card, and the electronic device can receive the BPF byte code sent by the operation and maintenance tool of the user through the DPU network card.
S120, updating a hardware flow table component of the DPU network card according to the BPF byte code.
In the embodiment of the disclosure, the electronic device may update the hardware flow table component of the DPU network card according to the BPF byte code.
Alternatively, the DPU network card may be computer hardware that communicates on an electronic device.
Optionally, the hardware flow table component is a data structure in the electronic device for how the data packets are handled and forwarded.
Specifically, the electronic device may update the hardware flow table component of the DPU network card according to the BPF bytecode after receiving the BPF bytecode.
S130, carrying out path matching processing on the received network data flow through the hardware flow table component to obtain a corresponding target path.
In the embodiment of the disclosure, the electronic device may perform path matching processing on the received network data flow through the hardware flow table component to obtain a corresponding target path.
Alternatively, the path matching process may be to determine that the network data stream is transmitted over a matched path.
Alternatively, the target path may be a path for transmitting the network data flow.
Specifically, after receiving the network data stream, the electronic device may perform path matching processing on the received network data stream through the hardware flow table component, that is, a path matched with the network data stream in the hardware flow table component, to obtain a corresponding target path.
And S140, transmitting the network data stream based on the target path.
In an embodiment of the disclosure, the electronic device may transmit the network data stream based on the target path.
Specifically, after determining the target path in the hardware flow table component, the electronic device may transmit the network data flow through the target path.
Therefore, in the embodiment of the disclosure, the BPF byte code issued by the operation and maintenance tool of the user can be received, then the hardware flow table component of the DPU network card is updated according to the BPF byte code, then the path matching processing is performed on the received network data flow through the hardware flow table component to obtain the corresponding target path, and finally the network data flow is transmitted based on the target path, so that the hardware flow table component of the DPU network card can be updated according to the BPF byte code issued by the operation and maintenance tool of the user, the corresponding target path is determined to be transmitted through the hardware flow table component after the network data flow is received, and therefore the hardware flow table component of the DPU network card is updated according to the operation and maintenance tool of the user is not required to be changed, the adaptation cost is reduced, the user operation is reduced, and the user experience is improved.
Optionally, after S110, the DPU network card hardware flow table unloading method may further include performing an interpretation process on the BPF byte codes by using a BPF interpreter to obtain a flow table rule.
In the embodiment of the disclosure, the electronic device may perform interpretation processing on the BPF byte code by using a BPF interpreter to obtain a rule of the flow table.
Alternatively, the BPF interpreter may be for interpreting the BPF bytecode.
Alternatively, the flow table rule may be a rule related to the packet transceiving case. For example, the flow table rule mainly relates to configuration of a flow table, a viewing method, management of the flow table, and the like.
Specifically, after receiving the BPF byte codes, the electronic device may perform interpretation processing on the BPF byte codes through a BPF interpreter in the DPU network card, that is, extract the flow table rule in the BPF byte codes, to obtain the flow table rule.
Optionally, S120 may specifically include registering the flow table rule with the hardware flow table component of the DPU network card.
In an embodiment of the disclosure, the electronic device may register the flow table rule with the hardware flow table component of the DPU network card.
Specifically, after obtaining the flow table rule, the electronic device may register the flow table rule in the hardware flow table component of the DPU network card, for example, store the configuration, the checking method, the management of the flow table, and the like of the flow table in the hardware flow table component of the DPU network card, where the source address, the destination address, the port number, and other parameters describing the characteristics of the data packet may be stored.
Optionally, the step S130 may specifically include receiving the network data flow, obtaining header information of the network data flow, and performing path matching processing in the hardware flow table component based on the header information to obtain a corresponding target path.
In an embodiment of the disclosure, the electronic device may receive the network data stream and obtain header information of the network data stream.
Alternatively, the header information may include a source internet protocol (Internet Protocol, IP) address, a destination IP address, a protocol type, a packet size, a port number, a sequence number, an acknowledgement number, a header length, a flag bit, and the like.
Specifically, after receiving the network data flow, the electronic device may acquire header information corresponding to the network data flow, for example, a source IP address, a destination IP address, a protocol type, a packet size, a port number, a sequence number, an acknowledgement number, a header length, a flag bit, and the like.
Further, the electronic device may perform path matching processing in the hardware flow table component based on the header information, to obtain a corresponding target path.
Optionally, performing path matching processing in the hardware flow table component based on the header information comprises searching whether the flow table rule corresponding to the header information exists in the hardware flow table component.
Specifically, the electronic device may obtain header information of the network data flow, and find whether the flow table rule corresponding to the header information exists in the hardware flow table component. For example, the source address, destination address, port number, and some other parameters describing the characteristics of the data packet are stored in the hardware flow table component, and the electronic device may look up whether the flow table rule corresponding to the header information exists in the hardware flow table component.
Optionally, S130 may specifically include determining that the target path is a fast path if the flow table rule corresponding to the header information is found in the hardware flow table component.
In the embodiment of the present disclosure, in the case that the flow table rule corresponding to the header information is found in the hardware flow table component, the electronic device may determine that the target path is a fast path.
Alternatively, the fast path may be a fast processing path preset according to a rule of the flow table.
Specifically, if the flow table rule corresponding to the header information is found to exist in the hardware flow table component, that is, the flow table rule corresponding to the related header information is already stored in the hardware flow table component, the electronic device may determine that the target path is a fast path, that is, transmit the network data flow through the fast path set in advance according to the flow table rule.
Optionally, S130 may specifically include determining that the target path is a default path if the flow table rule corresponding to the header information is not found in the hardware flow table component.
In the embodiment of the present disclosure, in a case where the flow table rule corresponding to the header information is not found in the hardware flow table component, the electronic device may determine that the target path is a default path.
Alternatively, the default path may be a preset general transmission path.
Specifically, if the flow table rule corresponding to the header information is not found in the hardware flow table component, that is, the flow table rule corresponding to the related header information is not stored in the hardware flow table component, the electronic device may use the target path as a default path, that is, transmit the network data flow through a preset general default path.
