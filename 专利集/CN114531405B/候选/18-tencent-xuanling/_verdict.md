## 最终判定

**第 4 档：公开资料不足（弱候选）**

第5档硬门槛核对：未取得任何针对玄灵的正向反向事实（无"玄灵仅单卡""玄灵明示用另一套机制""自有文档/自有专利写明另一手段"之类正向否定）；所检材料均为通用 OVS/LACP 科普而非玄灵产品级披露，属"公开资料未提及"，按规则**不算反向证据，只算公开资料不足**，故不落第 5 档。F# 字面命中 0 项、等同命中 0 项，<60% 命中 → 落第 4 档（弱候选），非强候选。

判定依据（基于上表 F# 分布）：玄灵为腾讯自研闭源芯片、公开度低，4 条 react 粗筛 query 均未检索到描述其实现本专利区分机制（跨 N≥2 块独立网卡 LACP 聚合 + 精确流表向全部网卡冗余下发以消除单网卡单点故障）的一手公开材料，亦无腾讯同主题专利；F1-F5 全部"公开资料不足"，但无任何正向反向事实，故判弱候选而非排除。
