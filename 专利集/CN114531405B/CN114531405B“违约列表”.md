## 3. 关键发现摘要

1. **零候选达到"确认侵权"档（第 1-2 档）**——在公开材料范围内**没有发现**任何候选明确实现 F1+F2+F3+F4 组合。本专利"两层端口聚合 + 跨 NIC 同步精确流表卸载"在公开技术栈中确为新颖发明点。
2. **F2 真反向证据集中**：Microsoft SET（switch-independent only 禁 LACP）、Red Hat（推荐 active-backup 非 LACP）、NVIDIA VF-LAG（intra-HCA only）、Intel ice driver（SR-IOV ⊥ LAG）、Cisco ACI Virtual Edge（明确不支持 LACP）—— 主流国际厂商在 NIC hw-offload 路径上**普遍走 active-backup / switch-independent / 不支持 LACP** 的设计偏好。
3. **F4 真反向证据集中**：VMware DSE（Active/Standby 或 vDS-isolated）、AWS Nitro（multi-card = independent ENI 不同 IP/subnet）、AMD-Pensando（vendor 自承无客户 use case）、阿里 Triton/Nezha（per-host AVS + 远端 idle pool 范式与 sync 思路相反）。
4. **闭源 + 公开度低的候选集中在 4 档弱候选**：国内云（CIPU 已排除、TGW、BVS、国资云）+ 国产 DPU 厂（中科驭数、YunSilicon 强候选、大禹、星云、星融元）。这些候选**法务取证升级 priority 高**——闭源不等于不侵权，需 NDA 渠道 / 反向工程升级。
5. **YunSilicon metaScale 是 6 国产 DPU 中证据最强**——Kube-OVN 官方文档公开 `BR_PHY_BOND_NAME=bond0` 配置参数 + `HW_OFFLOAD=true` + switchdev/SR-IOV 模式。但 LACP 模式未公开确认（建议核查 vendor datasheet / CCSA TC614 提交件）。

### 4.2 现有技术 caveat

- AccelNet NSDI 2018 paper（Firestone et al.）+ Andromeda NSDI 2018 paper 早于本专利申请日 2020-10-31。两篇均描述**单 SmartNIC** + slow-path fallback / encryption offload，**与本专利新颖点（跨 NIC 二层映射 + 同步卸载到 N 张 NIC）不重合**——不构成现有技术覆盖 ≥ 60%。
- Mellanox VF-LAG 早期 patch（2018-12，Saeed Mahameed）描述单 HCA 双口 LAG，同样与跨多卡的本专利新点不重合。
