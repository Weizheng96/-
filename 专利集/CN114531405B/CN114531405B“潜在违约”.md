# CN114531405B 潜在违约（初筛）

## 检索范围与时间
- **专利授权日**：2023-06-06
- **检索截止日**：2026-04-27
- **检索方式**：WebSearch（多组组合 query），涉及候选组织官网/官方文档、技术博客、SIGCOMM 2024 论文、CSDN/知乎/InfoQ 等技术社区、媒体报道。
- **核心 query 模板**：
  - `<组织名> DPU/智能网卡 OVS 卸载 多网卡 bond LACP <year>`
  - `<organization> SmartNIC dual NIC bond LACP OVS hardware offload`
  - `<organization> multi-DPU OR "two DPUs" bond OVS failover`
  - `<organization> 流表 同步 网卡 故障 切换`
- **必要技术特征对比基准**（来自专利介绍 F1-F6）：
  - F1 vSwitch 主体；F2 多 VM + N ≥ 2 NIC（注意：是 NIC，不是单 NIC 两 port）；F3 NIC 内部物理端口 LACP → 该 NIC 的逻辑端口；F4 vSwitch 把 N 个逻辑端口映射到同一目标端口标识（第一端口）；F5 慢路径触发后把同一精确流表"同时下发"到全部 N 张 NIC；F6 NIC 侧硬件 offload 流表存在。

> 重要技术判别：业内常见的 **NVIDIA VF-LAG / SR-IOV bond** 字面上只支持 **同一张物理 NIC 上的两个端口绑定**（NVIDIA Docs 明确说明），属于单 NIC 双 port 场景，**字面上不命中 F2**（专利明文 N ≥ 2 NIC，且 F3 要求每张 NIC 内部已有 LACP 聚合）。要落入保护范围，部署必须包含：(a) 至少两张物理 NIC/DPU 卡；(b) 每张卡内部的端口已 LACP 聚合；(c) 主机 vSwitch 把两张卡再聚合为同一上联端口；(d) 同一精确流表同时下发到两张卡。
