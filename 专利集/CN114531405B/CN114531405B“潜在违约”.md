## 关键发现摘要

1. **零候选达到"确认侵权"档（第 1-2 档）**：在公开材料范围内，**没有发现**任何候选明确实现 F1+F2+F3+F4 的组合。本专利的"两层端口聚合 + 跨 NIC 同步精确流表卸载"在公开技术栈中**确为新颖**。

2. **大量候选呈现 F2 真反向证据**：
   - Microsoft SET：明确"switch-independent only"，**禁** LACP
   - Red Hat：推荐 active-backup（非 LACP），balance-tcp 被建议**不使用**
   - NVIDIA VF-LAG：明确"intra-HCA only"，跨卡 LAG hardware offload 不存在
   - Intel ice driver：SR-IOV mutually exclusive with bonding
   - 即使 Cisco ACI Virtual Edge 也明确不支持 LACP

3. **大量候选呈现 F4 真反向证据**：
   - VMware DSE：dual-DPU 是 Active/Standby 或 vDS-isolated（非 sync）
   - AWS Nitro：multi-card = independent ENI 不同 IP/subnet（无 sync）
   - AMD-Pensando：vendor 自承"no customer use case requires > 1 DSC"
   - 阿里云 Triton/Nezha：per-host AVS + 单 CIPU + 远端 idle SmartNIC 池范式（与 F4 sync 思路相反）

4. **Intel IPU + IPDK + Google Andromeda（候选 20）+ YunSilicon（候选 23）+ 国资云 / 运营商云（候选 15）是最值得 Step 6 法务取证升级的对象**：
   - IPDK 字面 LACP 命中 + Google Andromeda 闭源
   - YunSilicon Kube-OVN 文档公开 `BR_PHY_BOND_NAME` 参数
   - 国资云强合规驱动 + 招标体量大 + 闭源

5. **架构层级不符（R-7）筛选有效**：5 类候选在 Step 4 预排除阶段已被排除，避免了 Step 5 浪费检索预算。
