### 核心技术候选（依 R-6 归并规则成立）

#### CT-1：OVS 上游 bond port + tc-flower offload 框架

- 核心技术：Open vSwitch 主线代码内置的 bond port 抽象（`ovs-vsctl add-bond <br> <bond> <slave1> <slave2>` + LACP mode 4 = 802.3ad）+ tc-flower offload 控制面把规则推给底层 driver；具体跨卡 / 全卡 HW offload 由底层 driver（mlx5 / ifcvf 等）完成
- 原创 / 权属：Linux Foundation Networking（项目宿主，非营利）+ Red Hat（IBM 子公司 NYSE: IBM，主 maintainer）+ Mellanox/NVIDIA（NASDAQ: NVDA）+ Intel（NASDAQ: INTC）+ Cisco（NASDAQ: CSCO）多 maintainer
- 主体类型：S1（上游开源）+ S5（多发行版分发）
- 关联独立权：权 1 / 23 / 36
- 检索关键词：`OVS bond port LACP 802.3ad`、`ovs-vsctl add-bond LACP`、`tc-flower hardware offload bond`、`ovs commit log bond`

#### CT-2：Linux kernel bonding driver + mlx5 LAG offload

- 核心技术：Linux kernel `drivers/net/bonding/` + `drivers/net/ethernet/mellanox/mlx5/lag/` —— 内核 LACP 802.3ad 协议 + 底层 driver LAG offload 实现
- 原创 / 权属：Linus / Linux Foundation + Mellanox/NVIDIA（mlx5 LAG 主要贡献方）+ Intel
- 主体类型：S1 + S5
- 关联独立权：权 1 / 23 / 36

## R-2 拓扑变体细分说明（强制 — 整数限定 N ≥ 2）

本专利 F1 含整数限定 N ≥ 2 + F2 卡内多端口 LACP + F3 跨卡聚合（两层结构）。按 R-2 拓扑变体细分规则，下列候选必须拆为多个拓扑变体分别 verdict：

- **NVIDIA VF-LAG**：
  - **T-A 单 NIC 双 PF**（NVIDIA QSG 默认描述）—— N=1，不满足 F1
  - **T-B 多 NIC + 卡内 VF-LAG + OVS 跨卡聚合**（推测拓扑）—— NVIDIA 未明示完整路径

- **Red Hat OSP**（依赖 NVIDIA VF-LAG）：同样拆 T-A / T-B

- **AMD Pensando / Intel IPU / Marvell OCTEON / 中国 DPU 创业**：vendor 公开度低，T-A 默认 vs T-B 跨卡 HA 部署 需分别评估

每个拓扑变体作为独立 `candidate_slug`（`<base-slug>-topology-A` / `-topology-B`），Step 5 独立跑 §A 19 类源 + §C 子 agent 复核。
