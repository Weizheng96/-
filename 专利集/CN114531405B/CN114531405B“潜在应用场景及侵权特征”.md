## 一、潜在应用场景（严格来自背景技术 — 不外扩）

### 场景 1：数据中心服务器虚拟化网络（虚拟交换机 + 多网卡流表卸载）
- **背景技术 verbatim 依据**：> "在计算虚拟化后，一台主机被虚拟化成多台虚拟机（Virtual Machine，VM），同一主机内的虚拟机通过虚拟交换机（Virtual Switch，VSw）……实现数据转发"；> "硬件卸载交换技术由于软硬协同机制复杂存在跨网络接口卡可靠性问题，只能实现单一网络接口卡内不同链路的可靠性，如果该网络接口卡故障，那么整个系统就故障了，存在单点故障的风险。"
- **场景说明**：数据中心 / 云服务器内，宿主机用虚拟交换机（vSwitch，典型如 OVS）为多台 VM 转发流量，并把流表卸载到智能网卡 / DPU 硬件加速；本专利针对"跨多块网卡做流表卸载 + 链路聚合冗余"以消除单网卡单点故障。
- **使用本专利的典型形式**：宿主机 vSwitch 软件 + 多块智能网卡 / DPU 的硬件流表卸载方案（含 LACP/bond 跨卡聚合 + 流表向全部网卡下发）。背景技术仅描述这一类数据中心虚拟化网络场景。

> 背景技术只实质描述了"数据中心虚拟化网络的多网卡流表卸载"这一个场景，故仅列 1 个场景（宁少勿编）。

## 二、独立权利要求 + 侵权特征 F1-F5

### 权 1 verbatim
> 一种流表处理方法，应用于虚拟交换机，所述虚拟交换机连接 M 个虚拟机和 N 个网络接口卡，M、N 为大于或者等于 2 的整数，其特征在于，所述方法包括：建立所述 N 个网络接口卡对应的 N 个逻辑端口的 N 个端口标识与一目标端口标识之间的映射关系，以将所述 N 个逻辑端口聚合为第一端口；其中，每个网络接口卡对应的逻辑端口是每个网络接口卡的物理端口基于链路汇聚控制协议 LACP 聚合形成的逻辑端口；在所述 N 个网络接口卡中的目标网络接口卡查询到所述目标网络接口卡中不存在与所述目标网络接口卡接收到的报文对应的卸载流表时，通过所述第一端口将精确流表卸载至所述 N 个网络接口卡。

### F1-F5 清单
- **F1（虚拟交换机 + M VM + N 网卡，M、N≥2）**：> "应用于虚拟交换机，所述虚拟交换机连接 M 个虚拟机和 N 个网络接口卡，M、N 为大于或者等于 2 的整数"
- **F2（N 逻辑端口聚合为第一端口的端口标识映射）**：> "建立所述 N 个网络接口卡对应的 N 个逻辑端口的 N 个端口标识与一目标端口标识之间的映射关系，以将所述 N 个逻辑端口聚合为第一端口"
- **F3（每网卡逻辑端口由其物理端口经 LACP 聚合形成）**：> "每个网络接口卡对应的逻辑端口是每个网络接口卡的物理端口基于链路汇聚控制协议 LACP 聚合形成的逻辑端口"
- **F4（目标网卡流表 miss 触发）**：> "在所述 N 个网络接口卡中的目标网络接口卡查询到所述目标网络接口卡中不存在与所述目标网络接口卡接收到的报文对应的卸载流表时"
- **F5（经第一端口将精确流表卸载至全部 N 个网卡）**：> "通过所述第一端口将精确流表卸载至所述 N 个网络接口卡"

### 其他独立权（共检出 7 项独权）
- 权 11（方法 / 物理机侧）：与权 1 同一机制，主体改为"物理机包含主机和 N 个网络接口卡"，并显式写"目标网络接口卡接收报文（来自物理交换机或目标虚拟机）"——与权 1 差异仅在主体表述与报文来源描述，F1-F5 实质相同。
- 权 23（装置）：权 1 的装置化（建立单元 + 卸载单元），F1-F5 实质相同。
- 权 33（物理机）：含权 23 装置的物理机，F1-F5 实质相同。
- 权 34（物理机：处理器+存储器+通信接口，执行权 1-10 或 11-22）：纯硬件载体权。
- 权 35（芯片系统，执行权 1-10 或 11-22）：芯片载体权。
- 权 36（计算机存储介质，执行权 1-10 或 11-22）：介质载体权。

> 权 34/35/36 为载体型独权，落入与否完全取决于其承载的权 1/11 方法是否被实践，故侵权判定统一以 F1-F5（权 1 方法）为准。

### 关键限定词与隐含约束
- **整数下界 M、N ≥ 2**：必须 ≥2 块网卡（N≥2）、≥2 台 VM（M≥2）——单网卡方案不落 F1（关键反向信号）。
- **跨网卡链路聚合（LACP）**：聚合必须是"跨多块物理网卡的物理端口"经 LACP 形成逻辑端口，再把这些逻辑端口聚合成第一端口——单卡内多端口 bond 不满足"跨网卡"。
- **流表向"全部 N 个"网卡下发**：F5 要求精确流表下发到所有 N 块网卡（冗余 / 防单点），而非只下发到收包的那一块网卡——这是与普通单卡 OVS 卸载的核心区别。
- **miss 触发的精确流表（exact-match）卸载**：F4 是"目标网卡查不到对应卸载流表"时才触发 F5，即 first-packet miss → 下发 exact flow——upcall/慢路径语义。
