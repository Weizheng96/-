# US8635616B2 潜在应用场景

> 前置：[`US8635616B2"领域适配".md`]

## 推演方法
F1-F9：BE/FE + VF software + VF device + DMA workflow。当前 2026-05-09，关注 (a) 公有云 vDPA 部署、(b) DPU + vDPA 整机、(c) NFV / 电信云 vDPA、(d) 容器 vDPA、(e) AI / GPU vDPA。

## 场景列表

### 场景 1：DPU + vDPA 公有云部署（性能关键）
- 描述：DPU 提供 SR-IOV VF + vDPA 框架；VM 看 virtio 接口性能直通
- 具名：**NVIDIA BlueField + mlx5_vdpa**、**AMD/Pensando + vDPA**、**Intel IPU + ifcvf vDPA**、**Marvell OCTEON + vDPA**、**中科驭数 / 云豹 / 星云 / 大禹 + vDPA**
- 命中：F1-F9（全部独立权）；**高商业价值**

### 场景 2：Linux kernel vDPA 上游框架（CT-1）
- 描述：Linux kernel `drivers/vdpa/` + `drivers/vhost/vdpa.c` 主线代码
- 具名：Linux kernel 上游、Red Hat（主要贡献方）
- 命中：F1-F9；**高商业价值**

### 场景 3：QEMU + vhost-vdpa 后端
- 描述：QEMU 集成 vhost-vdpa 后端，启用 vDPA 设备 → VM
- 具名：QEMU 上游、Red Hat（主要 maintainer）
- 命中：F1-F9

### 场景 4：DPDK rte_vdpa 框架
- 描述：DPDK 层提供 vDPA driver hooks
- 具名：DPDK 上游
- 命中：F1-F8

### 场景 5：商业发行版 RHEL / RHEL OpenShift / openEuler 内置 vDPA
- 描述：商业发行版默认 ship vDPA framework + driver
- 具名：Red Hat、Canonical Ubuntu、SUSE、openEuler、Anolis OS
- 命中：F1-F9

排序：场景 2（上游 CT-1）= 场景 1（DPU 集成）> 场景 5（发行版）> 场景 3 > 场景 4
