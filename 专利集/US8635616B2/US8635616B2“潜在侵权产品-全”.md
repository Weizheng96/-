> 归并说明：
> - **开源 vDPA 栈**（Linux 内核 `drivers/vdpa` + `vhost-vdpa` + QEMU virtio/vhost-vdpa + DPDK `rte_vdpa`）合为**一个"技术"条目**——它本体即完整实现 F1–F9，下游各 Linux 发行版/超融合（RHEL、Ubuntu、SLES、Oracle Linux、Anolis、Nutanix AHV）只是集成、无对数据通路的独立二次创新，故并入该技术条目的"应用案例"。
> - **DPU/SmartNIC 厂商**各有独立的 VF 硬件 + 自研 vDPA 驱动（VF 软件实例），保留为独立"产品"条目。
> - **公有云自研栈**各自独立，保留为独立"产品"条目。
> - 长尾新增：**Napatech**（FPGA SmartNIC，Link-Virtualization 向 Host 暴露 virtio-net + virtio-blk 硬件卸载，production-grade GA，挪威上市）——同抽象层，纳入。
> - 长尾剔除：**F5 Networks**（其 320% 吞吐 demo 是把 BIG-IP VNF 跑在 NVIDIA 硬件 vDPA 上——F5 是机制的下游集成者，非 FE/BE+VF 机制实现者，不同抽象层）；**UCloud 优刻得**（公有云，但其虚拟化 I/O 卸载用的是 NVIDIA vDPA，集成而非自研该机制，已由候选 02 覆盖）。

## 各候选简介

### 01-opensource-vdpa-stack
- 类型：技术
- 名称：开源 vDPA 栈（Linux kernel `drivers/vdpa` + `vhost-vdpa` + QEMU virtio-net/vhost-vdpa + DPDK `rte_vdpa`）
- 组织：Linux/QEMU/DPDK 社区（Red Hat 主导提出并合入主线，2020 并入内核主线）
- 命中 F#（初判）：F1,F2,F3,F4,F5,F6,F7,F8,F9（本体完整实现接收/发送通路、FE/BE 设备模型、VF 软件实例与 VF 设备一一对应、BE 绑定 vDPA 驱动、DMA 直通）
- 公开度：高（内核源码 + QEMU/DPDK 源码 + Red Hat 系列技术博客全公开）
- 一句话定位：本专利混合直通架构在产业界的事实标准开源实现，所有硬件厂商驱动挂靠其上。
- 应用案例（技术类）：RHEL / OpenShift Virtualization、Ubuntu Server、SUSE SLES、Oracle Linux(UEK)、Anolis OS（龙蜥）、Nutanix AHV（均基于 KVM 集成 vDPA，无数据通路独立二次创新）。

### 02-nvidia-mlx5-vdpa
- 类型：产品
- 名称：NVIDIA BlueField-2/3 DPU、ConnectX-6/7 NIC 的 `mlx5_vdpa` 驱动（DOCA hardware vDPA）
- vendor：NVIDIA (Mellanox)
- 命中 F#（初判）：F1,F2,F5,F6,F7,F8,F9
- 公开度：高（DPDK mlx5 vDPA 文档 + DOCA 文档 + in-kernel `mlx5_vdpa` 源码）
- 一句话定位：in-kernel vDPA 两大参考实现之一，硬件 VF + vDPA 驱动作为"VF 软件实例"绑定后端，数据面 DMA 直通 Guest。

### 03-intel-ifcvf-vdpa
- 类型：产品
- 名称：Intel IFCVF（FPGA 100G VF）vDPA 驱动 / Intel IPU
- vendor：Intel
- 命中 F#（初判）：F1,F2,F5,F7,F8
- 公开度：高（DPDK IFCVF 文档 + in-kernel `ifcvf` 源码）
- 一句话定位：in-kernel vDPA 另一参考实现，"IFCVF datapath is virtio ring compatible, works as a HW vhost backend"，与本专利 VF 直通 DMA 通路高度对应。

### 04-napatech-link-virt
