# US8635616B2 潜在使用组织

> 场景只有 1 个（云计算/数据中心的虚拟机 I/O 虚拟化，软件兼容性 + 硬件性能兼顾），内部分网络 I/O 与存储 I/O 两条已商用产品线。本专利的"FE/BE 半虚拟化设备模型 + BE 绑定空闲 VF 软件实例 + VF 设备直接 DMA 到/从 Guest 预分配缓存"混合直通架构，在产业界最直接对应的落地形态是 **vDPA（virtio/vhost Data Path Acceleration）** 及其等价的"硬件 virtio 卸载 / SR-IOV + 半虚拟化前后端混合"方案。
>
> **同抽象层过滤器**：仅枚举那些**自研该数据通路机制**（虚拟化栈/Hypervisor 的 FE/BE+VF 绑定框架，或 DPU/SmartNIC 的 vDPA 驱动 = VF 软件实例 + 硬件 VF）的玩家。纯做物理网卡贴牌、只做应用层而不碰 I/O 虚拟化数据面的名企不列。
>
> **排除专利权人自身**：华为（Huawei / 华为云 / 鲲鹏 / openEuler）是本专利权利人，其自用不构成侵权，不列为候选。

### 场景 1：云/数据中心虚拟机 I/O 虚拟化（混合直通：控制面半虚拟化 FE/BE + 数据面 VF 硬件 DMA）

**A. 虚拟化栈 / Hypervisor / 开源框架（实现 FE/BE 设备模型 + VF 软件实例绑定的控制面+数据面框架）**

- **Linux 内核 vDPA 子系统（upstream）**（开源框架；i 知名）— Linux 内核 `drivers/vdpa` + `vhost-vdpa`，是本机制的事实标准开源实现，所有厂商驱动挂靠于此 — 代表性已商用项目：vhost-vdpa、vdpa netlink 管理框架。
- **Red Hat（IBM 子公司）**（软件巨头；i 知名）— RHEL/KVM/QEMU 是 vDPA 的主要商业化推动者，vhost-vDPA 数据通路随 RHEL 出货 — 代表性已商用产品：RHEL、Red Hat OpenShift Virtualization、QEMU/KVM(vhost-vdpa)。
- **QEMU 社区**（开源框架；i 知名）— 实现 Guest virtio 前端 + 用户态/内核态 vhost 后端（BE），是 FE/BE 设备模型的参考实现 — 代表性已商用项目：QEMU vhost-vdpa / virtio-net 后端。
- **DPDK（Linux Foundation 项目）**（开源框架；i 知名）— `rte_vdpa` 框架 + 各厂商 vDPA PMD（mlx5、ifc），用户态实现 VF 软件实例与 DMA 通路 — 代表性已商用项目：DPDK vDPA driver 框架。
- **Canonical**（知名 Linux 厂商；i 知名）— Ubuntu Server / Ubuntu Cloud 内核集成 vDPA、随云镜像出货 — 代表性已商用产品：Ubuntu Server LTS。
- **SUSE**（知名 Linux 厂商；i 知名）— SLES 集成 KVM + vDPA 通路 — 代表性已商用产品：SUSE Linux Enterprise Server。
- **Oracle**（软件/云巨头；i 知名）— Oracle Linux / UEK 内核、Oracle VM/OLVM 集成 KVM 虚拟化 I/O — 代表性已商用产品：Oracle Linux (UEK)、Oracle Linux Virtualization Manager。
- **VMware（Broadcom 子公司）**（虚拟化巨头；i 知名）— vSphere/ESXi 自有 I/O 虚拟化栈（半虚拟化前端 VMXNET3 + SR-IOV passthrough + UPT/Enhanced Networking）— 代表性已商用产品：VMware vSphere / ESXi。
- **Microsoft**（软件/云巨头；i 知名）— Hyper-V 的半虚拟化 NetVSC/StorVSC（FE/BE 式 VMBus 通道）+ SR-IOV，Azure Accelerated Networking — 代表性已商用产品：Windows Server Hyper-V、Azure Accelerated Networking。
- **Nutanix**（知名超融合厂商；i 知名）— AHV（基于 KVM）超融合虚拟化平台，含虚拟化 I/O 数据面 — 代表性已商用产品：Nutanix AHV / AOS。
- **OpenAnolis 龙蜥社区（阿里主导）**（知名开源 OS 社区；i 知名）— Anolis OS 集成 vDPA / 虚拟化 I/O 加速、对接阿里云 DPU — 代表性已商用产品：Anolis OS。

**B. DPU / SmartNIC 厂商（提供硬件 VF + vDPA 驱动 = "VF 软件实例"，数据面直通）**

- **NVIDIA（Mellanox）**（芯片巨头；i 知名）— ConnectX-6/7、BlueField-2/3 DPU 的 `mlx5_vdpa` 驱动是 in-kernel vDPA 两大参考实现之一 — 代表性已商用产品：ConnectX-6/7 NIC、BlueField DPU + DOCA（hardware vDPA）。
- **Intel**（芯片巨头；i 知名）— IFCVF（FPGA 100G VF）的 `ifcvf` 驱动是 in-kernel vDPA 另一参考实现；IPU（Mount Evans/E2000）— 代表性已商用产品：Intel IPU、IFC vDPA 驱动。
- **AMD（Pensando）**（芯片巨头；i 知名）— Pensando DSC DPU，提供 virtio/vDPA 卸载数据面 — 代表性已商用产品：AMD Pensando DPU（Elba/Giglio）。
- **Marvell**（芯片大厂；i 知名）— OCTEON DPU 系列，支持 virtio-net 硬件卸载 — 代表性已商用产品：Marvell OCTEON 10 DPU。
- **Broadcom**（芯片巨头；i 知名）— Stingray / 高端 NIC，支持 SR-IOV 与 virtio 卸载 — 代表性已商用产品：Broadcom Stingray SmartNIC / Thor NIC。
- **中科驭数 YUSUR**（DPU 独角兽；ii 独角兽）— K2/K2-Pro DPU，主打云网络/存储 I/O 卸载，宣称支持 virtio/vDPA — 代表性已商用产品：YUSUR K2 DPU + KPU。
- **云豹智能 Yunsilicon**（DPU 独角兽；ii 独角兽）— 自研 DPU 芯片，面向云数据中心 virtio 数据面加速 — 代表性已商用产品：云豹 metalDPU / Xinghe DPU。
- **大禹智芯 DaYu**（DPU 独角兽；ii 独角兽）— Paratus 系列 DPU，云网络/存储卸载 — 代表性已商用产品：大禹 Paratus DPU。
- **星云智联 NebulaMatrix**（DPU 独角兽；ii 独角兽）— 自研 DPU/SmartNIC，virtio 硬件卸载 — 代表性已商用产品：NebulaMatrix DPU/SmartNIC。

**C. 公有云厂商（自研虚拟化 I/O 数据面，自有 DPU/虚拟化栈在生产环境运营）**

- **AWS（Amazon）**（云巨头；i 知名）— Nitro 系统把网络/存储 I/O 卸载到专用 Nitro 卡，Guest 见标准设备前端、数据面硬件直通 — 代表性已商用产品：AWS Nitro System（Nitro Cards / ENA / EBS）。
- **Microsoft Azure**（云巨头；i 知名）— Azure Accelerated Networking（基于 SmartNIC/FPGA 的 SR-IOV + 半虚拟化回退路径）— 代表性已商用产品：Azure Accelerated Networking。
- **Google Cloud**（云巨头；i 知名）— 自研虚拟化栈与 IPU/Titanium 卸载，gVNIC 半虚拟化前端 + 硬件数据面 — 代表性已商用产品：Google Cloud gVNIC / Titanium offload。
- **Alibaba Cloud 阿里云**（云巨头；i 知名）— 神龙（X-Dragon）+ CIPU，把虚拟化 I/O 卸载到自研 DPU，Guest 走 virtio 前端 — 代表性已商用产品：阿里云 CIPU / 神龙架构、弹性裸金属。
- **Tencent Cloud 腾讯云**（云巨头；i 知名）— 自研 DPU（银杉/玄灵）做虚拟化网络/存储卸载 — 代表性已商用产品：腾讯云 DPU / 黑石、CVM 虚拟化 I/O 加速。

> 跨场景兜底：本场景内部已细分网络 I/O 与存储 I/O 两条线；多业务集团（NVIDIA/Intel/AMD/各云厂商）在两条线均有产品，Step 4 据此对每条线分别展开具体产品。无第二个独立场景，故无需跨场景补列。

**Step 3 完备性自检**：
① 每个角色组（A 虚拟化栈框架 / B DPU 厂商 / C 云厂商）是否做了"技术相关玩家枚举"而非举几个代表？——已对三组分别枚举主流玩家。
② 横跨网络/存储两条线的集团是否在相关产品线均覆盖？——NVIDIA/Intel/AMD/各云厂商的 DPU 均同时承载网络与存储卸载，Step 4 分行展开。
③ 是否漏掉"做这件事"的知名主体/独角兽？——已覆盖 in-kernel 两大参考驱动（mlx5/ifcvf 对应 NVIDIA/Intel）、三大公有云自研卸载、主流 DPU 独角兽；华为作为权利人有意排除。
④ 反向核查：是否混入只是品类知名、并不实践本机制的名企？——未列入纯应用层 SaaS、纯物理网卡贴牌、与 I/O 虚拟化数据面无关的厂商。①②③ 通过，④ 已剔除无关层 → 进 Step 4。
