## Abstract

The patent provides virtualization methods optimizing performance and compatibility. A computing node includes hardware layers, a Host, and VMs. The I/O device virtualizes into VF (virtual function) devices with corresponding VF software instances. The Host contains a back-end instance (BE) of an I/O virtual device, while VMs contain front-end instances (FE). The BE binds with idle VF software instances, establishing architecture where each VM independently uses one VF device with high-efficiency interfaces and low latency overhead.

## Background + Summary

I/O virtualization 在 virtualization technology 中长期是难点。传统软件方案（如 vhost-net）兼容性好但性能损失；硬件方案（如 SR-IOV passthrough）性能好但兼容性差（与 guest OS 对接困难、缺少热迁移能力）。

本专利建立一个由 Host 中 BE 实例 + VM 中 FE 实例构成的 I/O virtual device 抽象层，与 SR-IOV VF + VF software instance 双向绑定 — 即 **vDPA (vhost Data Path Acceleration)** 架构原型：
- VM 看到的是 FE（标准 virtio 接口，提供兼容性）
- 数据路径直通 VF 硬件（提供性能）
- BE 通过 VF software instance 桥接控制面 + DMA 描述符传递

## 权利要求

### Claim 1（独立权·方法 — 含完整 DMA workflow）

A virtualization processing method applied to a computing node, the computing node including: a hardware layer, a Host running on the hardware layer, and at least one virtual machine VM running on the Host, where the hardware layer includes an input/output I/O device, several corresponding virtual function VF devices are virtualized from the I/O device, the Host has several VF software instances, the several VF software instances and the several VF devices are in one-to-one correspondence; the Host further has a back-end instance BE of an I/O virtual device having the same type with the I/O device, the VM has a front-end instance FE of the I/O virtual device; the BE in the Host is bound with an idle VF software instance; the method includes: pre-allocating, by the FE, a cache for direct memory access (DMA); acquiring, by the VF software instance bound with the BE, an address corresponding to the cache for DMA through an exporting application programming interface of the BE, writing the acquired address corresponding to the cache for DMA into a first storage unit of a VF device corresponding to the VF software instance; selecting, by the VF device, an address corresponding to the cache for DMA from the first storage unit when there is data to be received, and initiating a DMA write request by using the selected address corresponding to the cache for DMA as a target address; notifying, by the VF device, the VF software instance which is corresponding to the VF device and is in the Host after the DMA write request is executed, so that the VF software instance triggers the FE to receive data written into the cache corresponding to the address.

### Claim 11（独立权·方法 — 创建阶段）

A virtualization processing method, including: after an I/O virtual function of an input/output I/O device is enabled, generating several VF software instances in a Host; where several corresponding virtual function VF devices are virtualized from the I/O device with the I/O virtual function enabled; the several VF software instances and the several VF devices are in one-to-one correspondence; creating, by the Host, an I/O virtual device having the same type with the I/O device, where a back-end instance BE of the I/O virtual device is created in the Host, a front-end instance FE of the I/O virtual device is created in the initiated virtual machine VM; and binding the BE with an idle VF software instance.

### Claim 19（独立权·Host 装置）

A host, including: a first creating module configured to, after an I/O virtual function of an input/output I/O device is enabled, generate several VF software instances in the Host; where several corresponding virtual function VF devices are virtualized from the I/O device with the I/O virtual function enabled; the several VF software instances and the several VF devices are in one-to-one correspondence; a second creating module configured to create an I/O virtual device having the same type with the I/O device, where, a back-end instance BE of the I/O virtual device is created in the Host, a front-end instance FE of the I/O virtual device is created in the initiated virtual machine VM; and a binding module configured to bind the BE created by the second creating module with an idle VF software instance created by the first creating module.

### Claim 20（独立权·计算节点 / 整机）

A computing node, including: a hardware layer, a Host running on the hardware layer, and at least one virtual machine VM running on the Host, where the hardware layer includes an input/output I/O device, several corresponding virtual function VF devices are virtualized from the I/O device, the Host has several VF software instances, the several VF software instances and the several VF devices are in one-to-one correspondence; the Host further has a back-end instance BE of an I/O virtual device having the same type with the I/O device, the VM has a front-end instance FE of the I/O virtual device; the BE in the Host is bound with an idle VF software instance.

### Claim 21（独立权·计算机系统）

A computer system, which may include: At least one computing node [as in claim 20].
