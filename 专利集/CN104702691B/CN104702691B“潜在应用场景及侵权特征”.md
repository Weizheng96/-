## 一、潜在应用场景（严格来自背景技术 — 不外扩）

### 场景 1：有总控节点的分布式集群负载均衡
- **背景技术 verbatim 依据**：>"现有分布式系统中，以总控节点的有无来区分的话，一般有两种架构。一种架构中，分布式系统可包括客户端、存储节点和总控节点。"
- **场景说明**：总控节点（master）负责元数据管理、数据分布策略和集群管理的分布式系统，总控节点需在节点增减或故障时重新分配主备分区以实现负载均衡。
- **使用本专利的典型形式**：总控节点内的负载均衡调度算法/模块

### 场景 2：无总控节点（P2P）的分布式集群负载均衡
- **背景技术 verbatim 依据**：>"在另一种没有总控节点的分布式系统架构中，没有总控节点，通过在所有存储节点中选举出一个主(master)节点来完成总控的功能。在这种P2P系统中，元数据信息要么在所有节点上都有完整都一份，要么每个节点都有一部分元数据信息。"
- **场景说明**：通过选举产生master节点的P2P分布式系统，由被选举的master节点执行分区分配和负载均衡调度。
- **使用本专利的典型形式**：被选举master节点中执行的负载均衡逻辑

### 场景 3：分布式系统扩容（scale-out）时的负载均衡
- **背景技术 verbatim 依据**：>"分布式系统除了提供更强的处理能力，还需要能够弹性扩展和具有高可靠性。弹性扩展的能力就是集群可以灵活的scale-out(扩容)"
- **场景说明**：向分布式集群中新增节点后，需将原有节点上的部分主/备分区迁移至新节点以保持负载均衡，同时尽量减少数据迁移量。
- **使用本专利的典型形式**：扩容场景下触发的分区重分布调度算法

### 场景 4：分布式系统缩容（scale-in）时的负载均衡
- **背景技术 verbatim 依据**：>"当集群单点故障或缩容后，为保证数据均匀分布，需要重新调整主分区在各个节点上的分布情况，存在大量数据迁移操作。"
- **场景说明**：从分布式集群中移除节点后，需将被移除节点上的主/备分区重新分配到剩余节点，同时尽量减少数据迁移量。
- **使用本专利的典型形式**：缩容场景下触发的分区重分布调度算法

### 场景 5：分布式系统节点故障时的负载均衡
- **背景技术 verbatim 依据**：>"现有方案单点故障后数据负载将被多个节点分担，但是比较难保证数据负载被其他节点完全均匀分担，可能造成负载不均衡。"
- **场景说明**：集群中某节点（或某组节点）发生故障时，需将其主分区对应的备分区升级为主分区，并将故障节点的所有分区重新分配到其他节点，同时保持负载均衡。
- **使用本专利的典型形式**：故障场景下触发的备分区升级+分区重分布调度算法

## 二、独立权利要求 + 侵权特征 F1-Fk

### 权 1 verbatim
> 1.一种分布式负载均衡方法，应用于分布式系统，其特征在于，所述分布式系统的集群中包括多个节点和多个主分区，每个主分区对应于至少一个备分区，所述方法包括：
> 在所述多个节点上分配所述多个主分区，使得任意两个节点上的主分区个数差值小于第一预定阈值；
> 将所述多个主分区中的第一主分区对应的第一备分区分配到多个第二类节点中，使得所述第二类节点中任意两个节点上所述第一备分区个数之差都小于第二预定阈值，其中，所述第二类节点为所述第一主分区中第一类节点以外的节点，所述第一类节点为所述第一主分区所在的节点，或所述第一主分区所在的节点所属的分组的节点；
> 将所述多个第二类节点中的高负载节点的至少一个所述第一备分区移动到所述多个第二类节点中的低负载节点上，其中，所述高负载节点的分区个数大于第三预定阈值，所述低负载节点的分区个数小于第四预定阈值。

### F1-Fk 清单

- **F1（分布式系统前提）｜[AND]**：所述分布式系统的集群中包括多个节点和多个主分区，每个主分区对应于至少一个备分区
  - 可能描述方式：集群由N个节点组成/数据分片（shard/partition）具有M个副本（replica），其中一个为主副本（primary/leader）、其余为备副本（backup/follower/replica）/ 每个分片有1个主副本和至少1个备副本 / 采用多副本机制（replication factor ≥ 2），主副本提供服务

- **F2（主分区均匀分配）｜[AND]**：在所述多个节点上分配所述多个主分区，使得任意两个节点上的主分区个数差值小于第一预定阈值
  - 可能描述方式：将主分片（primary shard/partition）通过轮询（round-robin）/ 哈希（hash）分配到所有节点，使各节点主分片数之差不超过1 / 主副本在节点间均匀分布，任意两节点主副本数量差 ≤ 1 / 采用平均分配策略，各节点主分区数最多差1（或不超过K）

- **F3（备分区分配至第二类节点 + 均匀约束）｜[AND]**：将所述多个主分区中的第一主分区对应的第一备分区分配到多个第二类节点中，使得所述第二类节点中任意两个节点上所述第一备分区个数之差都小于第二预定阈值，其中，所述第二类节点为所述第一主分区中第一类节点以外的节点，所述第一类节点为所述第一主分区所在的节点，或所述第一主分区所在的节点所属的分组的节点
  - 可能描述方式：某个主分片对应的备副本（backup replica）分配到不包含该主分片的节点/不同机架/不同可用区（AZ）的节点上 / 备副本均匀分布在"非主副本所在节点（或节点组）"的其余节点上 / 各备副本在允许节点上的数量差不超过阈值 / 实现故障域隔离（fault domain），备副本不在同一节点/机架/AZ

- **F4（高负载→低负载移动备分区）｜[AND]**：将所述多个第二类节点中的高负载节点的至少一个所述第一备分区移动到所述多个第二类节点中的低负载节点上，其中，所述高负载节点的分区个数大于第三预定阈值，所述低负载节点的分区个数小于第四预定阈值
  - 可能描述方式：在备副本分布的节点集合内，将备副本从负载高（分区数超过上限）的节点迁移到负载低（分区数低于下限）的节点 / 根据节点负载/分区数进行rebalance，从overloaded节点搬移备副本到underloaded节点 / 触发重新均衡（rebalancing）时，在非主节点间移动备副本使各节点备副本数趋于均衡

### 其他独立权

- **权 7（装置/混合）**：与权1对应的装置权利要求，将步骤功能化为"第一分配单元"、"第二分配单元"、"备分区移动单元"三个模块。侵权特征F1-F4的语义与权1完全相同，仅执行主体从"方法步骤"变为"功能单元"。

### 关键限定词与隐含约束
- **整数限定**："第一/第二/第三/第四预定阈值"均为正整数；第一、第二阈值优选为1（差值≤1），第三/第四阈值基于平均值或自定义Max/Min
- **分组约束**：F3中"第一类节点"有两种定义方式（节点级或分组级），候选中二者任选其一即可满足——即备分区可避开同一节点，也可避开同一机架/可用区/节点组
- **多主分区前提**：集群中须存在"多个主分区"（≥2），方法针对每个主分区执行备分区分配
- **三层结构**：主分区均匀分配 → 备分区按主分区分散到第二类节点且均匀 → 第二类节点内高负载→低负载移动备分区，三步顺序执行
