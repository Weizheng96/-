## 一、潜在应用场景（严格来自背景技术 — 不外扩）

### 场景 1：分布式存储 / 分布式数据库的分区-副本均匀分布
- **背景技术 verbatim 依据**：> "分布式系统中的数据可以分成多个数据分片，每个数据分片称为一个分区。分布式系统中的一个节点可以拥有一个或多个分区。为了提高系统的可靠性和容错性，分布式系统往往会存储一份数据的多个副本，其中一个为主副本，其他为备副本，由主副本对外提供服务……分布式系统的负载均衡问题就转化为所有主分区和备分区在集群节点上均匀分布的问题。"
- **场景说明**：分布式存储系统 / 分布式数据库（含 NoSQL、NewSQL）把数据切成分区（shard / partition），每个分区有一个主副本 + 若干备副本，需要让主、备分区在集群所有节点上均匀分布以避免单点瓶颈。
- **使用本专利的典型形式**：内嵌于系统的分区分配 / 副本放置（replica placement）/ rebalance 模块；通常是数据库 / 存储引擎自带的核心调度逻辑。

### 场景 2：分布式缓存 / 内存数据网格（in-memory data grid）的分区与副本均衡
- **背景技术 verbatim 依据**：> "开源内存网格hazelcast提供了一种分布式系统中节点个数发生变化时的负载均衡方法。将分布式集群上数据的哈希空间分成多个等份，每一份就是一个分区，每个分区都有多个副本。分配主分区时，先采用Round-Robin方式将所有未分配的主分区分配到所有节点上，再从负载重的节点上移出多余主分区到负载低的节点上……"
- **场景说明**：内存数据网格 / 分布式缓存把哈希空间切成分区，每分区多副本；节点增减时需重算分区分布并迁移。背景技术把 hazelcast 明确列为现有方案（即本专利要改进的对象）。
- **使用本专利的典型形式**：缓存 / 数据网格产品内置的 partition service / migration 逻辑。

### 场景 3：集群弹性扩缩容（scale-out / scale-in）与单点故障恢复时的再均衡
- **背景技术 verbatim 依据**：> "弹性扩展的能力就是集群可以灵活的scale-out(扩容)和scale-in(缩容)，高可靠性的基本要求就是要保证集群没有单点故障。一个好的分布式系统，不仅要能处理节点扩容、缩容和节点故障等场景，同时需要保证扩容、缩容和节点故障后整个系统也能负载均衡……当集群单点故障或缩容后，为保证数据均匀分布，需要重新调整主分区在各个节点上的分布情况，存在大量数据迁移操作。"
- **场景说明**：分布式集群在扩容、缩容、节点故障三类拓扑变化下，需要在尽量减少数据迁移的前提下重新实现主/备分区均衡。本专利的核心卖点即"减少节点间数据迁移操作"。
- **使用本专利的典型形式**：集群管理 / 控制面（总控节点或 P2P master 节点）中的 rebalance / failover 调度算法。

> 说明：背景技术整体围绕"分布式系统的主分区+备分区均匀分布"这一主题，三个场景为同一机制在不同载体（存储/数据库、缓存/内存网格、扩缩容&故障恢复）上的体现，均有 verbatim 锚点，未外扩。

## 二、独立权利要求 + 侵权特征 F1-F3

### 权 1 verbatim（方法）
> 1.一种分布式负载均衡方法，应用于分布式系统，其特征在于，所述分布式系统的集群中包括多个节点和多个主分区，每个主分区对应于至少一个备分区，所述方法包括：
> 在所述多个节点上分配所述多个主分区，使得任意两个节点上的主分区个数差值小于第一预定阈值；
> 将所述多个主分区中的第一主分区对应的第一备分区分配到多个第二类节点中，使得所述第二类节点中任意两个节点上所述第一备分区个数之差都小于第二预定阈值，其中，所述第二类节点为所述第一主分区中第一类节点以外的节点，所述第一类节点为所述第一主分区所在的节点，或所述第一主分区所在的节点所属的分组的节点；
> 将所述多个第二类节点中的高负载节点的至少一个所述第一备分区移动到所述多个第二类节点中的低负载节点上，其中，所述高负载节点的分区个数大于第三预定阈值，所述低负载节点的分区个数小于第四预定阈值。

### F1-F3 清单
- **F0（前提 / 系统结构）**：> "所述分布式系统的集群中包括多个节点和多个主分区，每个主分区对应于至少一个备分区" — 系统由多节点构成，数据分为主分区，每个主分区至少有一个备分区（主副本对外服务 + 备副本容错）。
- **F1（主分区均匀分配）**：> "在所述多个节点上分配所述多个主分区，使得任意两个节点上的主分区个数差值小于第一预定阈值" — 把主分区分布到各节点，任意两节点主分区数之差 < 第一阈值（优选 1，即近乎完全均匀）。
- **F2（备分区按"第二类节点"约束均匀分配）**：> "将所述多个主分区中的第一主分区对应的第一备分区分配到多个第二类节点中，使得所述第二类节点中任意两个节点上所述第一备分区个数之差都小于第二预定阈值，其中，所述第二类节点为所述第一主分区中第一类节点以外的节点，所述第一类节点为所述第一主分区所在的节点，或所述第一主分区所在的节点所属的分组的节点" — 每个主分区的备分区只能放在"第二类节点"（即排除主分区所在节点本身、或排除其所属分组的节点），并使这些备分区在第二类节点间均匀（任意两节点之差 < 第二阈值）。**核心约束：备副本不与主副本同节点 / 同分组（反亲和 anti-affinity），且按主分区为单位均衡。**
- **F3（备分区高负载→低负载迁移）**：> "将所述多个第二类节点中的高负载节点的至少一个所述第一备分区移动到所述多个第二类节点中的低负载节点上，其中，所述高负载节点的分区个数大于第三预定阈值，所述低负载节点的分区个数小于第四预定阈值" — 在第二类节点范围内，把高负载节点（分区数 > 第三阈值）上的备分区搬到低负载节点（分区数 < 第四阈值），实现备分区在节点间二次均衡。

**拆分规则说明**：权 1 主体的三个并列动作（"在多个节点上分配主分区"／"将备分区分配到第二类节点"／"将备分区从高负载移到低负载"）各拆为 F1/F2/F3；F0 为前提性系统结构限定，单列。禁止合并、未意译。

### 其他独立权
- **权 7（装置）**：与权 1 一一对应的装置形式——第一分配单元（对应 F1）、第二分配单元（对应 F2）、备分区移动单元（对应 F3）。判定时与权 1 同套 F# 复用，仅"方法 vs 装置"形式差异（一个软件系统实现该方法即同时落入权 7 的装置范围）。

### 关键限定词与隐含约束
- **主/备分区二元结构**：必须区分主分区（primary，对外服务）与备分区（backup/replica，容错），每主分区 ≥1 备分区。仅有分区无主备之分（如无副本的纯 sharding）不落 F0。
- **整数阈值 K1/K2**：第一、第二预定阈值为正整数，优选取 1（任意两节点分区数差 ≤1，即尽可能完全均匀）。判定"均匀分布"目标即对应此限定，不要求字面出现"阈值"一词。
- **反亲和约束（第二类节点 / 第一类节点）**：备副本的放置排除主副本所在节点本身，或更强地排除主副本所在"分组"（机架 / 可用区 / rack-awareness / zone-awareness）。这是 F2 的核心区别特征——很多系统都有"副本不同节点 / 不同机架"的 anti-affinity 策略。
- **两级均衡 + 减少迁移**：F1（主均衡）→ F2（备按主分组约束均衡）→ F3（备高低负载二次微调）的三段式，目的是在扩缩容 / 故障时最小化数据迁移。从属权 4/5/6 进一步覆盖 scale-out / scale-in / 节点故障三类触发场景。
- **高/低负载判定**：以"分区个数"是否超过 / 低于阈值（或平均值）界定高/低负载节点，而非以 CPU/流量等运行时指标——这是 F3 的限定方式。
