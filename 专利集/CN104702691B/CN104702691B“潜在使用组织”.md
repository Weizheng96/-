# CN104702691B 潜在使用组织

> 本专利是"**数据分区 + 主/备副本放置 + 再均衡**"算法（非网络/流量负载均衡）。因此只枚举**自研分区-副本放置 / rebalance 调度**的系统厂商（与 F1-F3 同抽象层）。
> **明确排除**（不同抽象层）：L4/L7 网络流量负载均衡器（F5、Nginx、HAProxy、Envoy、阿里 SLB / AWS ELB 等）——它们均衡的是请求流量而非数据分区副本，不落 F0-F3。
> 仅收录 (i) 知名组织 / (ii) 独角兽；标注同抽象层依据。许多产品横跨多场景，按"多业务跨场景兜底"规则在各相关场景分别列。

### 场景 1：分布式存储 / 分布式数据库 / 流式数据系统（分区 + 主备副本均匀分布）

> 涵盖 NoSQL / NewSQL / 分布式 SQL / 对象存储 / 消息流（leader-follower 分区副本）——均把数据切成分区、每分区主副本+备副本、需在节点间均匀放置且副本反亲和。

- **Elastic NV**（i 知名；NYSE: ESTC）— 搜索/分析引擎龙头 — 代表性产品：Elasticsearch（primary/replica shard + shard allocation awareness 机架/可用区反亲和 + rebalance）

### 场景 3：集群弹性扩缩容（scale-out/in）与单点故障恢复时的再均衡

> 此为"触发条件"型场景（拓扑变化下最小化数据迁移），由上述场景 1/2 的**同一批系统**在扩容/缩容/故障时执行其 rebalance/failover 逻辑实现——不重复另立新主体，候选与场景 1/2 一致（从属权 4/5/6 分别对应 scale-out / scale-in / 故障三类触发，取证时重点核查这些产品的 rebalance/failover 文档）。
