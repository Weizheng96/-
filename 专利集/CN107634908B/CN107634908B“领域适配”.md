## 领域分类

### 1. 主领域

- **主**：通信 / 云 / 网络（更具体：**实时音视频传输 + 自适应 FEC（前向纠错）冗余调度 + 拥塞控制**）
- **次（强相关）**：音视频 / 直播 / 流媒体；云游戏 / 远程渲染；VR/AR 实时云渲染

## Step 3c — R-STANDARD 子流程

### 1. 判定标准类别 + 查相关数据库

**候选标准 1 — IETF RFC 5109 / RFC 8627 (Generic RTP-FEC) + RFC 6363 (FEC Framework)**
- 类别：国际事实标准（IETF），但**非强制**——RTP-FEC 是可选 payload type
- IPR 状态：IETF IPR disclosures 数据库（[ietf.org/ipr](https://www.ietf.org/ipr/)）—— 检索"Huawei + FEC"应做 best-effort

**候选标准 2 — 3GPP TS 26.346 (MBMS FEC) + 3GPP TS 38.214 (5G NR FEC for transport block)**
- 类别：3GPP 行业标准——SEP 声明制度，IPR 数据库公开
- 但本专利的"业务类型识别"与"自适应 FEC 调度"**超出 3GPP TS 26.346 / TS 38.214 标准化 FEC 编码格式范围**——3GPP 只标准化 FEC 编码 / 解码格式，调度策略由 vendor 实现自定义
- ETSI IPR database 可检索 Huawei 在 3GPP SA4 / RAN1 中是否声明本族系为 SEP

**候选标准 3 — SMPTE ST 2022-1/-7 (Pro-MPEG FEC for broadcast)**
- 类别：广播流媒体行业标准——专业广播 / 视频生产领域
- 与本专利保护范围重合度低（SMPTE 2022-1 仅指定 row/column XOR FEC 编码格式，调度策略不在标准范围内）

**候选标准 4 — CCSA / 国家标准 / 行业准入**
- 本国候选：GB/T、YD/T 涉及"实时音视频通信""5G 云游戏"的国标 / 通信行业标准（YD/T 由 CCSA 制定）—— 主 agent 可通过 `ccsa.org.cn` 关键词检索"自适应 FEC""实时业务""差异化 QoS""5G 云游戏"等命中相关 YD/T 标准

### 2. 本专利与标准的关系判定

| 标准候选 | 关系判定 | 对 Step 4 候选清单的影响 |
| --- | --- | --- |
| RFC 5109/8627/6363（RTP-FEC） | **范围相邻、不完全重合** —— 标准只规定 FEC payload 编码格式，本专利保护"业务类型 + 网络状态 + 时延要求三因子自适应调度逻辑"超标准范围 | 候选清单不变；FEC payload 编码本身落入相邻方案（不命中 F1/F2/F3 同时） |
| 3GPP TS 26.346 / TS 38.214 | **范围相邻、不完全重合** | 同上 |
| SMPTE ST 2022-1/-7 | **范围相邻、不完全重合** | 同上 |
| 国内 YD/T / CCSA 标准 | **best-effort 未查明** | 标 "标准合规状态需法务确认"；按常规侵权流程处理 |
