## 候选简介选录

- **01-tencent-trtc / 02-tencent-meeting / 03-tencent-gme**：腾讯云音视频体系，公开有 ARS（自适应抗丢包）+ ULPFEC + 业务场景配置 SDK 接口，与 F1-F5 在原理层全面接近；公开度由文档丰富度决定。
- **05-aliyun-rtc / 07-bytedance-rtc / 08-agora-rtc / 09-netease-yunxin / 10-zego-rtc**：同类自适应抗丢包 RTC 引擎；其中 Agora 公开文档最丰富（SD-RTN 论文 + 招股书披露），适合作为深查样本。
- **11-zoom-meetings / 12-cisco-webex / 13-ms-teams / 14-google-meet**：海外视频会议大厂，公开技术细节较少但论文 / 专利 / RFC 贡献较多，是潜在长尾扩展的源头。
- **15-cmcc-onenet / 16-ctelecom-ctwing**：运营商 IoT 平台，下行控制消息通道常见 NB-IoT / Cat-1 链路抗丢包加固。
- **17-baidu-apollo / 18-banma-alios / 19-nio-cloud / 20-tesla-cloud**：车云通道 / OTA 下发场景，公开度低（多 NDA），但 Apollo 部分开源。
- **21-dji-ocusync / 22-xag-control / 23-xiaomi-cyberdog**：无人机 / 机器人控制链路，OcuSync 有公开白皮书。
- **24-epic-eos / 25-valve-sdr / 26-unity-relay / 27-riot-direct**：游戏低延时传输基础设施。Valve SDR 有官方技术博客，Unity 部分开源。
- **28-meta-airlink / 29-nvidia-cloudxr / 30-pico-connect**：VR 串流，对丢包鲁棒性高度敏感。NVIDIA CloudXR 有白皮书。
- **31-aliyun-cen / 32-tencent-ccn / 33-aws-global-accel / 34-cloudflare-argo**：DC 间智能路由 / 加速产品，命中场景 3。注意：本类产品大多基于路径选优 + QoS 不一定主动 packet-level FEC，需 Step 6 验证是否真有冗余包逻辑；Cloudflare Argo 文档丰富，作首选样本。
- **35-rootcloud / 36-siasun-rtmw**：工业互联网 / 机器人远程运维。
- **37-baijiayun-brtc**：教育 RTC 直播。
- **38-tech-webrtc-fec**：开源技术，是大量 RTC 产品的下游集成基础——Step 5 会把若干产品归并到这条技术下。
- **39-nvidia-geforce-now / 40-ms-xcloud / 41-google-stadia**：云游戏传输栈，专利时间窗 (2021-06 之后) 内三家都仍在运营或刚下线，证据可追。
