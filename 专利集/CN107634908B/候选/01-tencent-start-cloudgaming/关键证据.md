#### F2 — 根据网络状态变量、传输成功率、业务类型获取冗余包数量

| 项 | 内容 |
|---|---|
| 证据原文 | "针对网络随机丢包的情况，START 网络团队提出了**重传和 FEC 相结合**的可靠传输机制 Hairpin。Hairpin 利用在边缘节点部署服务器带来的低延迟的网络环境，对数据包、重传和冗余包进行多轮传输的最优组合，**通过实时监测网络状况，包括 RTT 和丢包事件，动态调整 FEC 参数**，以适应网络条件的快速变化。" |
| 网页链接 | [腾讯高校合作 NSDI 2024 文章](https://ur.tencent.com/article/1481) + [Hairpin GitHub 仓库（hkust-spark/hairpin）](https://github.com/hkust-spark/hairpin) |
| 命中类型 | **字面命中** — 这是与权 1 F2 "**根据预先获取的网络状态变量**（RTT / 丢包率）**、预先获取的传输成功率**（多轮传输最优组合内含传输成功率推断）**和所述业务类型**（cloud gaming 业务），**获取所述待传输数据包需要传输的冗余数据包的数量**" **几乎逐字对应**的实施描述 |

#### F3 — 根据时延要求获取冗余数据包传输总时间

| 项 | 内容 |
|---|---|
| 证据原文 | "云游戏对于画质、延迟、卡顿极为苛刻的要求……以零排队为目标的拥塞控制算法 Pudica，适用于对延迟和吞吐率有严格要求的交互式流媒体场景，该算法通过避免引入帧级别的排队来保证低延迟。" + AUGUR "**实现了卡顿率数量级的降低**……Wi-Fi + Cellular 双链路"——deadline 在 frame 级别 |
| 网页链接 | [Pudica USENIX NSDI '24 论文](https://www.usenix.org/conference/nsdi24/presentation/wang-shibo) + [AUGUR USENIX NSDI '24 论文](https://www.usenix.org/conference/nsdi24/presentation/zhou-yuhan) |
| 命中类型 | **字面命中**（frame-level deadline = 时延要求；零排队 = 传输总时间 ≤ 时延要求；从属权 3 字面命中）|
