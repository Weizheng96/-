# 22-xag-control verdict

## 候选基本信息（专利公开日 2021-06-08）

- **候选名**：极飞 XAG 农业无人机控制（飞控 / 数据链 / 集群通信子系统）
- **组织**：广州极飞科技股份有限公司（XAG）
- **类型**：产品 / 设备族（SuperX RTK 飞控、DL1 Data Link 数据链、Local Network Terminal、ARC3 Pro 远控器等）
- **初判命中 F#**：F2、F3、F4、F5（来自 Step 5 表格）
- **公开度**：低（XAG 未公开通信协议规范 / SDK / 开发者文档）
- **时间窗符合性**：极飞相关产品族（DL1 Data Link、SuperX RTK、V40/P100/V50 Pro 等）均在 2021-06-08 之后仍在大规模在售并不断推出新机型（V50 Pro 2023、V50 2022、P100 2022、P200 2026 等），**时间窗满足**——即"若有侵权事实，时间上不会因为产品早于专利公开日而被排除"。
- **场景定锚**：本候选所对应的 Step 2 应用场景中**没有"农业无人机"或"集群作业"** —— 专利说明书实施例 1 列举的场景为：IoT 用户控制消息、在线游戏、DC 间保活、机器人控制（注：泛指"工业/服务/远程手术/远程驾驶"等控制信令）、VR/全息体感、在线视频/视频会议。农业无人机定点喷洒 / 集群协同**不在专利说明书显式列举的应用场景内**——只能通过 "机器人控制 / 远程操控" 这条做最宽泛的类比，**场景吻合度本身就弱**。这是排在公开度问题之前的一条独立弱化因素。

## 最终判定 **第 4 档：公开资料严重不足，无法判定**

### 评级理由

- **公开资料严重不足** — Phase 1 共 5 个 query、Phase 2 共 10 次 fetch / curl，**未取得任何 packet-level 协议技术陈述**。XAG 不公开 SDK、不公开协议白皮书、不公开开发者文档；唯一的官方 "protocols" 站点实际是用户协议合同站，不是通信协议站。这一情形不是工具临时受限，而是 **XAG 本身不公开此层信息**。
- **F1-F5 全部 0 命中，但 0 命中 ≠ 反向证据** — 不能据此落 5 档（已排除）。未检索到任何 XAG 文档**明确**说 "不使用主动冗余 / 仅使用 ARQ / 固定调度间隔"。
- **场景吻合度本身偏弱** — 农业无人机不在专利说明书显式列举的应用场景内（实施例 1 的 "机器人控制" 边界覆盖力存疑）。
- **关键合理怀疑** — 农业无人机数据链中飞控指令、RTK 修正流、遥测回传、图传 / 视频等业务**天然分通道**传输，更可能走 "应用层直接指定业务类型"（=权 1 的从属权 4 形态），而非权 1 限定的 "发送端自动从流量统计识别业务类型"。但该怀疑同样**没有公开证据**支撑，因此不能作为 5 档排除依据。

## 升级路径（若后续要将判定从 4 档动起来）

- **路径 A — 升 3 档（疑似命中部分 F#）**：找到 XAG 公开的（白皮书 / 学术合作论文 / 招聘 JD / 开发者大会演讲 / 专利文献）任一来源，明示其数据链采用 "按业务类型自适应调整冗余包数量"。优先方向：(1) XAG 的中国专利公开文本（CNIPA 检索 "广州极飞" + "冗余" / "前向纠错" / "业务类型自适应"）；(2) XAG 校招 JD 中的 "飞控通信 / 数据链工程师" 技术栈描述；(3) 极飞研究院 / 高校合作论文。
- **路径 B — 升 2 档（命中独立权利要求至少 4 个 F#）**：在 A 基础上再确认时延要求驱动总时间（F3）+ 网络状态参与调度选择（F4）。
- **路径 C — 降 5 档（已排除）**：找到 XAG 文档**明确**陈述 "数据链使用 ARQ / 仅有链路层固定 FEC / 不区分业务类型" 等任一反向证据；或者 XAG 已经在与华为的某专利交叉许可 / 规避声明中公开规避了 CN107634908B。
- **路径 D — 抽取 FCC PDF**：候选目录已落盘 FCC 用户手册 PDF（2.1MB），后续可用 pdfplumber / OCR 进一步抽取技术参数（频率、调制方式、信道编码等）；若手册仅含合规性测试参数（FCC 通常如此）而无协议层算法，仍维持 4 档；若意外披露了 FEC / 自适应冗余设计，则按 A/B 路径升档。

## 总结一句话

XAG 极飞农业无人机控制候选时间合规、场景边缘（农业无人机不在专利显式场景，仅能经 "机器人 / 远程操控" 宽泛类比），5 个 query + 10 次抓取均**未取得任何 packet-level 协议技术陈述**也**未取得任何反向证据**，**落第 4 档（公开资料严重不足，无法判定）**。
