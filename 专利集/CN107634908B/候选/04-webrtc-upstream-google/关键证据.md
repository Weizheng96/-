| 项 | 内容 |
|---|---|
| 证据原文 | Pacer 优先级排序："The pacer ... uses round-robin distribution among equal-priority streams" + "**prioritizes packets by type (audio > retransmissions > video/FEC > padding)**" |
| 网页链接 | [WebRTC Pacer index.md](https://webrtc.googlesource.com/src/+/HEAD/modules/pacing/g3doc/index.md) |
| 命中类型 | **字面命中** — type priority 即业务类型差异化（audio = 高优先 + 不同 FEC 强度）|

| 项 | 内容 |
|---|---|

| 网页链接 | [api/fec_controller.h](https://webrtc.googlesource.com/src/+/master/api/fec_controller.h) + [Video coding in WebRTC](https://webrtc.googlesource.com/src/+/HEAD/modules/video_coding/g3doc/index.md) + [flexfec_sender.cc](https://webrtc.googlesource.com/src/+/1743a19183e65c338db05d2b212afd6a13151721/webrtc/modules/rtp_rtcp/source/flexfec_sender.cc) |
| 命中类型 | **字面命中** — FecController + FlexFEC + ProtectionBitrateCalculator = F2 完整链路 |

### F3 — 时延要求

| 项 | 内容 |
|---|---|

### F4 — 调度方法

| 项 | 内容 |
|---|---|
| 证据原文 | "The pacer employs a **leaky bucket algorithm** to regulate packet transmission. Rather than sending packets immediately, it queues media and releases them at calculated intervals based on the target send rate" |
| 网页链接 | [Pacer index.md](https://webrtc.googlesource.com/src/+/HEAD/modules/pacing/g3doc/index.md) |
| 命中类型 | **等同命中** — leaky bucket algorithm 等同于从属权 2 "均匀时间调度方法" |
