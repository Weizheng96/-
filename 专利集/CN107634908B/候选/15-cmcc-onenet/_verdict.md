## 最终判定 **第 5 档：已排除**

判定依据：
- **F1-F5 全部 0 命中**，且有 OneNET 官方协议文档作为**真反向证据**（不是"未检索到"的弱信号）；
- OneNET 公开采用 MQTT / CoAP / HTTP / LwM2M / 自研 EDP (TCP-based) 五大行业标准协议，**全部基于 ACK + 重传 (ARQ) 机制**，与权 1 核心要求的"主动冗余 packet-level FEC + 业务类型自适应调度"在**架构层完全不一致**；
- 平台官方首页 + EDP 规范双源证实"无 FEC / 无前向纠错 / 无主动冗余包 / 无业务类型自适应冗余调度"；
- EDP 文档明示"事务机制需要在上层实现"——把可靠性责任明确推给上层应用 (典型 ARQ 委派模式)，进一步说明平台层无主动冗余动作；
- 权属抗辩**不成立**（华为是真实权利人，非中国移动），但**架构反向证据**已足以单独支撑"已排除"。
